# Sense Style AI — персональный стилист на основе психологии моды

Веб-сервис: ответы диагностического квиза + фото клиентки → **Формула стиля**, измеримый
**разрыв идентичности (Identity Gap, %)** и капсула фотореалистичных образов на её собственном фото.

> Конкурсный MVP (Junior ML Contest, ИТМО). Источники правды:
> продукт — [`architecture/2026-04-27-mvp-spec.md`](architecture/2026-04-27-mvp-spec.md),
> метод — [`architecture/sense-style-method.md`](architecture/sense-style-method.md).

## Задача

Женщины 30–50 в точках перехода (карьера, материнство, эмиграция, смена статуса) сталкиваются с
разрывом: внутренне человек уже изменился, а внешний образ остался от прежней версии себя. Отсюда
«полный шкаф и нечего надеть».

Существующие AI-стилисты рекомендуют вещи по трендам и предпочтениям. Мы решаем другую задачу:
**измеряем разрыв между тем, как клиентку считывают сейчас, и тем, какой она хочет быть,** и
собираем гардероб, который этот разрыв закрывает.

Опора научная, не эзотерическая: **Self-Discrepancy Theory** (Higgins, 1987) и
**Enclothed Cognition** (Adam & Galinsky, 2012), плюс авторская методология стилиста
(4 базовых стиля → 25 подстилей, 7-шаговый алгоритм Формулы стиля).

## Пайплайн

```
Квиз Identity Scan ─┐
 (черты now / want) │
                    ├─→ POST /api/analyze
Фото в полный рост ─┘        │
                             ├─ analyze_photos → Vision: цветотип, контраст, геометрия фигуры
                             ├─ RAG retrieval  → правила авторской базы под ранние сигналы
                             ├─ diagnose       → Формула стиля + Identity Gap % + объяснимость
                             └─ directions     → 2 образа-превью на фото клиентки (бесплатно)
                                    │
                             /card  ▼
                             ├─ refine_substyle       → психотип (Big Five) уточняет подстиль из 25
                             ├─ generate_card_palette → 30 цветов + стоп-цвета
                             ├─ generate_capsule      → базовая капсула + образы по сценариям жизни
                             └─ render_look_on_client → примерка: её лицо и фигура в новых образах
```

**Ключевое инженерное решение:** Identity Gap считается серверным `diagnose()` **один раз** и
одинаково виден на всех экранах (квиз, Карта, трекер). Глубина платных уровней — в решении
(подстиль, капсула, психотип), а не в новом числе.

**RAG.** Авторская база правил (`data/rag/`) подмешивается в промпт диагностики до запроса и
прикрепляет «сработавшие правила» после — это даёт объяснимость. Семантический поиск
(`fastembed`) опционален: без ML-модели система мягко деградирует до поиска по тегам.

## Архитектура

| Модуль | Назначение |
|---|---|
| `app/main.py` | Flask-приложение: квиз, Карта стиля, кабинет, «брать/не брать», метрики |
| `core/pipeline.py` | пайплайн: vision → диагностика → палитра/капсула → рендер образов |
| `core/prompts.py` | загрузка системных промптов из `architecture/prompts/*.md` |
| `core/provider.py` | клиент OpenRouter (OpenAI-совместимый): chat, vision, генерация изображений |
| `core/rag.py` | retrieval авторских правил (fastembed + фолбэк на теги) |
| `core/config.py` | маршрутизация моделей по тирам (dev/final), ключ из окружения |
| `core/tracking.py` | события, воронка, история Identity Gap «до/после» |
| `core/catalog.py` | подбор вещей под палитру и стоп-цвета |
| `web/` | статика: лендинг, квиз Identity Scan |

Интеллектуальное ядро — **промпт-библиотека** в [`architecture/prompts/`](architecture/prompts/):
каждый промпт возвращает строгий JSON. Код — тонкая оркестрация поверх неё.

## Модели (через OpenRouter)

| Шаг | Модель | Почему |
|---|---|---|
| Vision, диагностика, палитра, капсула | `google/gemini-2.5-flash` | быстро (~8 с), стабильный JSON |
| Eval / «думающий» режим | `google/gemini-2.5-pro` | reasoning; для продуктовой генерации не используется — медленно и обрезает JSON |
| Превью-образы | `google/gemini-2.5-flash-image` (Nano Banana) | дёшево |
| Примерка на клиентке | `google/gemini-3-pro-image-preview` (Nano Banana Pro) | identity-preserving: держит лицо и фигуру |

Тир переключается через `SENSE_MODE=dev|final`. Провайдер сменяем за `core/provider.py` — пайплайн
не трогается.

## Запуск

### Docker (рекомендуется)

```bash
docker build -t sense-style .
docker run -p 8080:80 -e OPENROUTER_API_KEY=sk-or-v1-... sense-style
# http://localhost:8080   ·   проверка живости: /healthz
```

### Локально

```bash
pip install -r requirements.txt
export OPENROUTER_API_KEY=sk-or-v1-...        # Windows: «Переменные среды»

python -m flask --app app.main run --port 5000 --debug
```

Опционально — семантический RAG (иначе работает фолбэк на теги):

```bash
pip install fastembed numpy
python -m scripts.build_rag_index
```

## Тесты

```bash
pytest -q      # 66 тестов, без обращения к внешним API
```

Покрыто: сквозной путь квиз → Карта (единство Identity Gap, отсутствие петли при перезапуске),
инвалидация кэша Карты при новой диагностике, гарантия «ровно 6 образов», подбор по палитре,
фолбэк RAG без ML-модели, трекер Gap.

CI (GitHub Actions): тесты + сборка Docker-образа + smoke-тест `/healthz` и наличия статики в образе.

## Метрики

Приватная страница `/metrics` (доступ по email основателя или `?key=SENSE_METRICS_KEY`):
воронка (просмотр формы → сборка Карты → сгенерированные образы → отзыв), динамика Identity Gap,
собранные отзывы. События пишутся через `core/tracking.py`.

## Безопасность и данные

- Ключ OpenRouter — только в переменной окружения / `.env` (в `.gitignore`). В коде ключа нет.
- Фотография обрабатывается эфемерно и **удаляется после генерации**; в базе остаются результаты
  и история Identity Gap.
- Данные пользователей РФ хранятся на территории РФ; передача фото AI-обработчику — по отдельному
  согласию. Подробнее — страница `/privacy`.

## Статус

MVP работает end-to-end: квиз → диагностика → Карта стиля с образами на фото клиентки → трекер
разрыва → проверка вещи «брать/не брать». Планы разработки — в [`plans/`](plans/).
