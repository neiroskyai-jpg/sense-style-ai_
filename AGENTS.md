# AGENTS.md

## Что это за проект

**Sense Style AI** — персональный AI-стилист с Vision-анализом для женщин 30–50 в точках перехода. Подход проекта — psychology-first: анализ одежды строится через психологический профиль клиентки.

## Источники правды

- Главный источник правды по продукту: `architecture/2026-04-27-mvp-spec.md`
- По методологии: `architecture/sense-style-method.md`
- По визуальному направлению: `architecture/visual-direction.md`
- Если `.business/` противоречит `architecture/`, приоритет у `architecture/`
- Бизнес-контекст и стратегия: `.business/INDEX.md`

## Что читать перед задачей

- Всегда сначала читать `CLAUDE.md`
- Для продуктовых и технических задач затем читать `architecture/2026-04-27-mvp-spec.md`
- После этого читать только те документы из `.business/` и `plans/`, которые нужны для текущей задачи

## Правила работы с `plans/`

- `plans/` содержит технические планы отдельных функций
- Один план = одна функция
- Если план для функции уже существует, работать нужно с ним
- Имя файла плана: `YYYY-MM-DD-название-функции.md`
- План должен быть разбит на фазы со статусами `[ ]` или `[x]`
- В конце плана должен быть итог: что реализовано и что осталось
- После каждой сессии план нужно актуализировать

## Правила работы

- Перед началом работы проверять `git status` и `git diff`
- Не переписывать работающий код без необходимости
- Не выполнять `git commit`, `git push`, публикацию и деплой без отдельного разрешения пользователя
- Не удалять и не перемещать файлы без предварительного объяснения
- Не открывать и не обрабатывать содержимое `user-photos/` и `voice-samples/`, если это не требуется текущей задаче
- Перед существенными изменениями сначала описать план и дождаться подтверждения пользователя
- После изменений запускать релевантные тесты и сборку
- Не раскрывать секреты из `.env`
- Всегда отвечать пользователю на русском языке

## Завершение сессии

- В конце каждой сессии записывать итог по правилам из `CLAUDE.md` в `.business/история/YYYY-MM-DD-краткое-название.md`
