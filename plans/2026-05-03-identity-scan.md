# План: Identity Scan — диагностика Формулы стиля

**Дата создания:** 2026-05-03
**Назначение:** двухуровневая диагностика идентичности и Формулы стиля. Лид-магнит на лендинге + полная диагностика для платных тиров. Заменяет текущий маркетинговый прототип `.business/assets/identity-scan-quiz.html` на честную реализацию метода.
**Источник правды по методу:** [`architecture/sense-style-method.md`](../architecture/sense-style-method.md), разделы 3 (5 этапов) и 7 (7-шаговый алгоритм Формулы стиля).

## Контекст

Существующий прототип квиза не реализует методику: выдаёт «тип разрыва» (хаос/страх/идентичность/система), которого нет в методе, и не считает базовый стиль по 4 семантическим полям. Нужна переработка.

**Архитектура двух уровней:**
- **Identity Scan** — публичный лид-магнит, 12 вопросов, 3–5 мин. Выдаёт приближённый Identity Gap % и гипотезу о доминирующем поле (например, «лидерство Классики с нотой Драмы»).
- **Identity Scan Pro** — полная диагностика после регистрации/оплаты, 10–13 мин. Реализует все 5 этапов метода и 7-шаговый алгоритм. Выдаёт Формулу стиля + визуальную формулу + 2–3 референса.

**Стек пилота:** HTML/Vanilla JS (тот же файл, что сейчас). Перенос на Next.js — после валидации метода на 4 живых клиентках.

## Принципы UX-сокращений (для Pro)

Чтобы уложиться в 10–13 мин без потери методологии:
- **40 ключевых характеристик** вместо 80 (по 10 на семантическое поле — Натуральный/Романтика/Драма/Классика). Метод сохраняется.
- **Один проход вместо двух.** Для каждого слова 3 кнопки: «считывают сейчас» / «хочу, чтобы считывали» / «не моё». Дальше из этого считается разрыв.
- **Тренды и love/stop — визуально.** Галерея из 10–12 образов с реакцией «нравится / не моё / стоп».
- **Фото и параметры — отдельный шаг с прогресс-баром «загрузка профиля»**, не воспринимается как «тест».

### Фаза 2 · Identity Scan Pro · полная диагностика без Vision + Supabase

Файл: новый `.business/assets/identity-scan-pro.html` (или раздел в существующем). Vision-автоматика откладывается — на этой фазе стилист сама вводит визуальные параметры по фото клиентки. **Параллельно подключаем Supabase** — это решение, как сохранять ответы из обоих квизов (лид-магнита и Pro), без отдельных хранилищ типа Formspree.

- [ ] **2.1** **Этап 1 метода — Контекст и роли.** 5 экранов: распределение Work/Home/Self в %, ЦА восприятия (для кого одеваешься в каждой сфере), дресс-код среды, контекст перемен, возраст.
- [ ] **2.2** **Этап 2 метода — Имидж-диагностика.** Сетка из 40 характеристик (10 × 4 семантических поля). Каждое слово размечается одной из 3 кнопок. После — экран «3 главные характеристики желаемого впечатления» (выбор из помеченных «хочу»). Свободное поле для самоощущения.
- [ ] **2.3** **Этап 4 метода — Технические данные (ручной ввод).** Параметры: рост, размер одежды, размер обуви. Тип фигуры — выбор из 5 (с визуальными подсказками-силуэтами). Тональные характеристики — подтон / глубина / контраст (3 select). Загрузка 2 фото (портрет + полный рост) — пока без анализа, только для просмотра стилистом.
- [ ] **2.4** **Тренды + love/stop — визуально.** Галерея 10–12 образов: «нравится / не моё / стоп». Ценовой сегмент — slider или select (low / mid / high / open).

- [ ] **2.6** **Экран результата.** Формула стиля (комбинация подстилей × нота), визуальная формула (силуэт / ткани / цвета / длины / стоп), 2–3 референса с фото и обоснованием по 2 осям. Возможность экспорта в PDF (для пилота — `window.print()`).

## Что валидируем на пилоте (Фаза 2.8)

Главный вопрос пилота: **даёт ли 7-шаговый алгоритм правильную Формулу стиля на живой клиентке.**

Критерий успеха для Микуловой: алгоритм выдаёт `Power Woman × Авангардная нота × Drama-акцент` (см. раздел 11 метода) при вводе её реальных характеристик. Если выдаёт что-то другое — баг в алгоритме или в подстановке параметров.

Для остальных трёх клиенток успех = стилист (Ксения) согласна с результатом по 2 осям (впечатление + физика).
