# План: MVP-срез для конкурса (вертикальный пайплайн)

**Цель:** запускаемый MVP в публичном репозитории к 20 июля 2026 (3-я волна Junior ML Contest).
**Темп:** 5 ч/день. Окно: 25 июня – 20 июля (подача), защита 27–29 июля.
**Связан с:** [2026-06-25-itmo-jmlc-zayavka.md](2026-06-25-itmo-jmlc-zayavka.md)

## Что строим — один сквозной сценарий

```
квиз Identity Scan → Gap-профиль
   + фото вещи
        → vision-analyzer.md   → Claude Vision (анализ вещи)
        → formula-diagnostic.md → диагностика стиля (Формула стиля)
        → look-generator.md    → генерация капсулы (изображение)
        → результат на экране + Identity Gap %
```

Промпты уже готовы в `architecture/prompts/` — MVP это обёртка вокруг них, а не разработка с нуля.

## Стек (под критерии «Инженерия» + «DS»)

**Решение (2026-06-25):** конкурсный MVP строим на **Python** (не Next.js из CLAUDE.md).
Причина — жюри сильнее всего смотрит на DS и инженерию ML-пайплайна, промпты уже
написаны под Python, eval на Python собирается естественно. Next.js-продукт по
CLAUDE.md — после конкурса. Отклонение зафиксировать в mvp-spec.

- **AI-слой:** пакет `core/` — загрузка промптов из `architecture/prompts/`, клиент
  OpenRouter (OpenAI-совместимый), vision + диагностика. Фото сжимаются до 1024px.
- **Eval (Python):** `eval/` — ground-truth + метрики (accuracy Vision vs эксперт,
  табу-цвета, A/B генераторов).
- **Фронт:** лёгкий — переиспользуем HTML-квиз + простая страница результата (FastAPI/статика).
- **Инфра:** Docker, GitHub Actions CI (pytest), README с архитектурой и метриками.

## Структура репозитория (реализовано)

```
core/
  config.py     — маршрутизация моделей по тирам (dev/final), ключ из env
  prompts.py    — загрузчик системных промптов (2 формата) + кэш
  provider.py   — клиент OpenRouter, vision, сжатие фото, парсинг JSON
  pipeline.py   — vision → диагностика → (капсула)
scripts/run_vision.py — CLI первого живого вызова
tests/          — pytest загрузчика (без API), зелёные
eval/           — метрики (Фаза 3)
Dockerfile, .dockerignore, .github/workflows/ci.yml, README.md, .env.example
```
