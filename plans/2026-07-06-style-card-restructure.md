# Карта стиля: чёткий путь клиента + картинки вещей

**Дата:** 2026-07-06
**Статус:** в работе

## Зачем

QA-прогон фаундера на себе (`клиенты/тест Ксения`, 51 скрин + 2 PDF) вскрыл: Gap-синхрон работает
(78% везде), образы-примерки на клиентке реалистичны и в тоне — но **путь клиента размыт**:

1. **Дубли.** Одно и то же по 2–3 раза:
   - Фигура/силуэты: «Твоя основа» + «Посадка под фигуру» + «Силуэты под фигуру» (3 блока об одном).
   - Психология: «Почему именно этот подстиль» + «Твоя натура и стиль» (2 блока).
   - Перечень вещей: «Базовая капсула» + «Капсулы под жизнь» + «Два направления».
   - Анкета линий/посадки: в анкете Карты и снова в конструкторе «Брать/не брать».
2. **Непоследовательный визуал.** Образы на клиентке — фото (хорошо), цвета — плашки (хорошо),
   а вещи базовой капсулы — только текст. Запрос фаундера: **картинки самих вещей**.
3. **Баги.** PDF: 8 пустых страниц + пустой блок «ОБУВЬ». «Брать/не брать»: вердикт `чем заменить: null`.
   Кабинет-демо показывает чужие данные (Анна/18% при Ксении/78%).
4. **Нет ясной оси.** Карта — простыня из ~12 секций вперемешку.

Канон спеки (`architecture/2026-04-27-mvp-spec.md`): тариф «Стилевая карта» = Формула + палитра +
6 образов под сценарии с лицом клиентки. Ядро — «увидеть **себя** в образах»; капсула словами выросла
сверх спеки и конкурирует с образами за внимание.

## Решения фаундера (2026-07-06)

- **Картинки вещей** — из **каталога-packshot** (реальные фото из партнёрских фидов, кликнуть и купить),
  не AI-генерация. См. связку с планом кабинета Фазы 8–10 (`plans/2026-07-02-subscription-cabinet-capsules-seasons.md`).
- **Порядок работ** — сначала **структура + дедубль** (без API, каркас), потом картинки вещей.

## Целевая структура Карты (ось «кто ты → твои правила → гардероб → что купить»)

1. **Кто ты** — Формула + Gap + психопортрет (объединить «Почему подстиль» и «Твоя натура» в один блок).
2. **Твои правила** — Цвет (цветотип → палитра → стоп-цвета) и Фигура (один блок: тип + посадка + силуэты).
3. **Твой гардероб** — базовая капсула → образы под сценарии (фото) → стилизация.
4. **Что купить** — шопинг-лист со ссылками → ориентир → стоп-лист.

## Фазы

### Фаза 1 — Дедубль + 4 блока (STYLE_CARD template) [x]
- Слиты два психо-блока («Почему подстиль» + «Твоя натура») → один «Почему это твой стиль».
- Фигура (основа + посадка + силуэты) сведена в один блок «Твоя фигура — что носить»; силуэты — один
  раз (figure_fit.silhouettes или фолбэк c.silhouettes). Цветотип перенесён подводкой к палитре.
- Добавлены навигационные разделители 4 блоков (`.blocklead`): 01 Кто ты · 02 Твои правила ·
  03 Твой гардероб · 04 Что купить. Заодно определён отсутствовавший CSS `.meta`.
- Проверка: рендер шаблона с фейком OK, дубли = 0 (проверено count по заголовкам), 62/62 тестов. Без API.

### Фаза 2 — Картинки вещей в базовой капсуле (каталог-packshot) [ ]
- **Формат уточнён по референсу Style DNA (2026-07-06):** базовая капсула = **коллаж-раскладка образа
  из packshot-вещей** на белом с тегом-сценария (Работа/Выход/Повседневное), как «Outfit ideas» у
  Style DNA — вместо текстовых карточек. Вещи из наших партнёрских фидов (кликнуть → купить).
- **Наше отличие от Style DNA:** рядом с коллажем — фото клиентки в этом образе (примерка = наше УТП,
  «это ты, а не модель»). У них коллажи обезличенные; у нас и вещи, и человек в них.
- Упирается в точность подбора и packshot-съёмку (Фазы 8–10 плана кабинета). Фолбэк — текущий текст.
- **НЕ копировать:** брендинг «AI Stylist» (у нас текст без «AI»), кислотно-розовую палитру
  (у нас Petrogradka Editorial — приглушённый глянец).
- **БЛОКЕР (проверено 2026-07-06, 0 API):** текущий каталог не тянет коллаж.
  - 40 товаров, 2 бренда узких стилей: Lichi (платья/сатин/кружево = Романтика), Ushatava
    (органза/авангард = Драма). Нет деловой базы (жакеты-власти, строгий низ).
  - Цвета каталога (бежевый/голубой/молочный/сиреневый/шоколад) ≠ палитры большинства формул.
  - Подбор под Power Woman × Smart Casual выдал голубые платья/шоколадные комплекты — мимо палитры
    и формулы. Коллаж на этом = нерелевантные вещи, хуже текста.
  - Фото на моделях, не packshot.
  - **Вывод:** Фаза 2 не делается на текущих данных. Нужно решение фаундера (см. развилку ниже).

### Развилка Фазы 2 (данные изменили прошлый выбор «каталог-packshot»)

  команды отклоняются). Первый прогон может требовать доводки (WB хрупкий: dest/appType, basket-хосты картинок).
  После сбора — подключить `products_wb.csv` к каталогу и построить раскладку.
- **B. AI-генерация packshot** (переиграть прошлый выбор): рисуем вещи капсулы на белом по описанию

  + расход ~$0.4–1.2/карта. Годится для демо-визуала ИТМО.
- **C. Пока оставить текст**, вернуться к коллажу после расширения каталога.

### Фаза 5 — Editorial-оформление Карты (типографика + крупные фото) [x] (нужна визуальная проверка фаундером)
- Подключены Cormorant Garamond (заголовки) + Onest (текст) вместо Georgia. `h1` 54px, `h2`/formula 30px,
  названия образов серифом 23px — глянец вместо «документа».
- Контейнер расширен 760→820px (фото образов крупнее), больше воздуха (padding/margins), лид `.dna` 18px.
- Проверка: рендер OK, Cormorant применён (4 места), 62/62 тестов. Без API. Смотреть в `/card?text=1`.
- **Осталось при желании:** ещё крупнее подача образов (журнальные развороты в одну колонку) — по фидбеку.

### Фаза 6 — Порядок секций Карты по цепочке клиента [x] (визуальная проверка фаундером)
- Решение фаундера 2026-07-06: порядок **Gap → образы → фигура → цветотип → цвета → гардероб → купить**
  (образы = вау, показать сразу; разбор — после). Переставлено в STYLE_CARD:
  01 Кто ты → 02 Твои образы → 03 Твоя фигура → 04 Твои цвета → 05 Твой гардероб → 06 Что купить.
  Макрос `lookcard` поднят в начало (иначе Jinja не находит). Проверено: порядок по позициям верный,
  образы выше палитры, 62/62 тестов.

### Фаза 7 — Повтор квиза → чистый Gap (не липнет старый результат) [x]
- Решение фаундера: при новом квизе НЕ показывать старую Карту. `/card`: если `_card_stale` — ведём на
  форму пересборки с плашкой `notice` «Твой разрыв обновился: N%». Только Gap переходит из квиза,
  Карта собирается заново. Добавлен стиль `.notice` в CARD_BUILD_FORM, событие `card_stale_rebuild_prompt`.

### Фаза 8 — Устранение петли квиз↔Карта + рассинхрон + цена [x]
- **Рассинхрон Gap (квиз 44% ≠ Карта 78%)**: баг `_card_stale` — старые Карты без отпечатка считались
  «не устаревшими» → показывался прежний Gap. Фикс: при отсутствии отпечатка сверяем Gap напрямую.
- **Квиз липнул старый результат**: убрано авто-восстановление готового результата при заходе на квиз
  (`web/identity-scan-quiz.html`) — теперь чистый старт всегда, кроме продолжения незавершённой генерации.
  `?fresh=1` явно показывает интро.
- **Петля квиз→Карта→квиз**: диагноз анонимного квиза жил только в памяти `_JOBS`, терялся при рестарте
  сервера (особенно с `--debug`) → `/card?from_job` не находил диагноз → редирект на квиз. Фикс:
  `_save_pending_diag`/`_load_pending_diag` — диагноз на диск (`data/pending_diag/`), переживает рестарт.
- **Цена премиума 35 000 → 15 000 ₽** (первая работа, дешевле): лендинг `web/index.html` (старая 35 000
  зачёркнута) + спека (маржа пересчитана 91%→79%). ВНИМАНИЕ: 15 000 ≈ средний тариф спеки (14 900) —
  тарифную сетку стоит пригладить отдельно (на лендинге среднего нет, визуально ок).
- Проверка: 63/63 тестов, pending-diag round-trip, компиляция. Требует проверки фаундером на живом
  сервере (`--debug` + Ctrl+Shift+R).

### Фаза 3 — Баги [~]
- **PDF-разрывы [x] (нужна проверка фаундером в реальном html2pdf).** Причина белых листов: `legacy`
  pagebreak-mode (плодил пустые страницы) + 2-колоночная сетка образов ломала разбивку. Фиксы:
  - `@media print`: одна колонка (`.looks/.caps`), карточки не режутся (`break-inside:avoid`),

    `print-color-adjust:exact` (фоны/палитра печатаются), скрыт блок фидбека.
  - `downloadPdf`: убран `legacy` mode → `mode:['css','avoid-all']`; `windowWidth:820`.
- **Осталось:** «Брать/не брать» `чем заменить: null`; пустой блок «ОБУВЬ» (пустые карточки капсулы —
  это данные, не PDF); кабинет-демо с чужими данными (Анна/18%).
