# Цветовые пары в палитре: цвет → состояние

**Дата:** 21.07.2026
**Источник идеи:** `architecture/competitor-content-teardown.md`, схема 3 («пара с ярлыком-состоянием»).

## Проблема

Панель «Твоя палитра» в Карте ([app/main.py:1205](../app/main.py)) показывает квадратики цветов
и молчит. Клиентка видит 12 оттенков и не знает, что с ними делать: какие два взять вместе,
что это ей даст. Палитра — самый «научный» на вид блок Карты и при этом самый бесполезный.

У Носачевой ровно этот контент (пара цветов + ярлык «элегантно / современно / женственно»)
собирает 15–58% сохранений. Но у неё ярлык произвольный. **У нас есть то, чего у неё нет:
want-профиль из диагностики.** Значит пара подписывается не абстрактным «элегантно», а тем
состоянием, которое клиентка сама назвала в квизе. Это палитра, пристёгнутая к Identity Gap.

## Решение

5–7 пар из её палитры, каждая — с ярлыком-состоянием и коротким «зачем», плюс реверс ролей
(одна пара = два образа). Ярлыки берутся не из головы модели, а маппятся на 4 семантических
поля Формулы (`semantic_field_distribution`): drama → «заметно, с характером», classic →
«собранно, статусно», romantic → «мягко, женственно», natural → «спокойно, по-своему».

Приоритет пар — по разрыву: сначала те, что закрывают поле с наибольшим `want − now`.

## Фазы

### [ ] Фаза 1. Генерация пар
- `core/pipeline.py`: функция `generate_color_pairs(diagnosis)` рядом с `generate_styling_pair`.
- Вход: палитра (`hex` + `name` + `group`), `semantic_field_distribution`, `now_field_distribution`,
  `stop_colors`, цветотип.
- Выход: `{"pairs": [{"a": {...}, "b": {...}, "state": "...", "why": "...", "roles": ["...", "..."]}]}`
- Ограничения в промпте: не брать цвета из стоп-листа; в паре один цвет — нейтраль, второй — акцент
  (или две нейтрали для базовой пары); ярлык состояния — из словаря полей, не выдуманный;
  `why` — 1 фраза через психологию, на «ты», без восклицаний.
- Лёгкий focused-вызов (flash), как `generate_styling_pair`, ~1200 токенов.

### [ ] Фаза 2. Рендер в Карте
- Панель `#palette` ([app/main.py:1205](../app/main.py)): блок «Пары, которые работают на тебя».
- Каждая пара — два кружка цвета + ярлык состояния + фраза «зачем».
- Реверс показывается текстом: «сверху A + снизу B — …; поменяй местами — …».
- Не ломать существующий вывод групп (base / main / accent), встать под ним, до ссылки «Как
  использовать палитру».

### [ ] Фаза 3. Связка с капсулой
- В капсуле образы должны опираться на эти же пары, а не на свои. Прокинуть выбранные пары
  в `generate_capsule` как ограничение (правила 11–14 уже вшиты в `_CAPSULE_QUALITY_RULES`).
- Проверить, что образ не собирается из цветов, которых нет в парах.

### [ ] Фаза 4. Контент-повод
- Тот же вывод — готовый формат рилса для аккаунта Ксении: «пара цветов + состояние».
- Отличие от конкурентов в подписи: у них «элегантно», у нас «работает на запрос "чтобы слышали"».
