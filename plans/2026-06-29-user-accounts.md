# План: Аккаунты пользователей (magic-link) + персональный профиль

**Цель:** рабочий сервис с регистрацией — для тестирования и демонстрации жюри. Вход по
magic-link (без паролей), персональный профиль (анкета «Примерочной» + Формула + история
Identity Gap) сохраняется на сервере и доступен с любого устройства.

**Стек:** Flask-сессии (подписанный cookie), SQLite на постоянном томе Amvera (`/data`,
через `SENSE_DATA_DIR`), magic-link на `itsdangerous` (самоподписанный токен, без таблицы
токенов), отправка почты — Unisender (РФ, 152-ФЗ), с dev-фолбэком (ссылка на экране).

**Связан с:** [2026-06-25-mvp-vertical-slice.md](2026-06-25-mvp-vertical-slice.md), Примерочная (`/garment`).

## Решения

- **Magic-link, не пароли** — нет хранения паролей (меньше 152-ФЗ-нагрузки), современный UX.
- **Вход опционален** — гость пользуется как сейчас (localStorage); вошедший получает
  серверный профиль и кросс-девайс.
- **Почта — Unisender (РФ)** вместо Resend (США) — данные в РФ. Только для отправки ссылки.
- **Хранилище — SQLite на `/data`** (том Amvera). Управляемый PostgreSQL не нужен на этом объёме.

## Предусловия

- [x] `SENSE_DATA_DIR` — **больше не нужен в панели**: `config.data_dir()` сам берёт `/data` (persistenceMount Amvera), если смонтирован.
- [x] `SENSE_SECRET_KEY` — **не обязателен**: `config.secret_key()` генерирует стабильный секрет в файл на томе (не в git). Можно переопределить env'ом.
- [ ] `UNISENDER_API_KEY` + `UNISENDER_FROM_EMAIL` — для реальной отправки писем (без них — dev-фолбэк: ссылка на экране). **Нужно от фаундера.**
- [ ] Уточнить продукт Unisender: транзакционный (Unisender Go / UniOne) vs классический — от этого зависит эндпоинт (`UNISENDER_API_URL`).
