# План: AI-генерация 2 фото клиентки в её формуле стиля (внутри квиза)

**Дата:** 2026-05-19
**Триггер:** запрос Ксении — на финальном экране квиза клиентка должна увидеть **свои фото в её формуле стиля** (вместо референс-фото других девушек, как сейчас).

## Цель функции

Закрыть Identity Gap **в момент максимального вовлечения** — сразу после квиза. Клиентка проходит диагностику, видит свою формулу (например, «Драма × Классика»), и тут же **видит себя в этой формуле**. Это резко повышает шанс покупки тарифа «Карта стиля» (3 900 ₽).

**Почему ВНУТРИ квиза, а не на отдельной странице:** см. memory `project_wow_in_quiz`. Раньше по MVP-спеке это был шаг `/upload` через email после квиза — но email-воронка теряет 60-70% людей. WOW-эффект работает только если показать прямо сейчас.

## Архитектура

```
[Квиз 14 шагов] → [Шаг 15: «Загрузка фото в полный рост»] → [Спиннер 2-5 мин: «ИИ создаёт твоё преображение»] → [Финальный экран: 2 фото клиентки в её формуле]
                              │
                              ├─→ Supabase Storage (сохранить оригинал фото)
                              │
                              └─→ Vercel Edge Function /api/generate-styling
                                        │
                                        ├─→ Claude Vision API (анализ лица, фигуры, цветотипа)
                                        ├─→ Replicate API (Nano Banana) ×2 — генерация двух образов
                                        └─→ Возврат 2 URL фото
```

## Стек

- **Бэкенд:** Vercel Edge Function (роут `/api/generate-styling`). Можно и Supabase Edge Function — Vercel проще, так как сайт уже там.
- **Анализ фото:** Claude Vision (`claude-sonnet-4-6`), один запрос на анализ.
- **Генерация:** Replicate API, модель Nano Banana (быстрее Midjourney, ~30-90 сек/фото). Альтернатива — Flux Pro, если Nano Banana по качеству не дотянет.
- **Промпт для генерации:** «Photorealistic full-body portrait of [описание лица из Claude Vision] wearing [одежда из формулы стиля], [сценарий], [цветовая палитра из методологии], editorial photography, soft daylight, neutral background.»
- **Хранение:** Supabase Storage. Бакет `user-photos` (приватный, RLS по user_id). Поля: `original_photo_url`, `generated_photo_1_url`, `generated_photo_2_url`.

## Какие 2 образа генерировать

**Открытый вопрос** — нужно согласовать с Ксенией. Варианты:

1. **Главное направление + нота.** Например, «Драма × Классика» → 1 фото «больше Драма», 2 фото «больше Классика».
2. **2 сценария.** Например, «деловая встреча» (закрывает запрос) + «свидание» (показывает гибкость формулы).
3. **ДО → ПОСЛЕ.** 1 фото оригинал (без обработки), 2 фото с AI-стайлингом по формуле.

**Рекомендация:** вариант 1 («формула в двух проявлениях»), потому что он лучше демонстрирует методологию.

## Бюджет на одну прогон

| Что | Цена |
|---|---|
| Claude Vision (1 анализ) | ~$0.30 |
| Replicate Nano Banana × 2 | ~$0.20 |
| Supabase Storage (10 МБ × 6 фото) | копейки |
| **ИТОГО за 1 клиентку** | **~$0.50** |

При 100 клиентках/мес = $50, при 1000 = $500. Контролируемо.

## Защита от злоупотреблений

- **Rate limit:** 1 генерация на email/IP в 24 ч.
- **Капча или email-верификация ДО шага загрузки** (иначе кто-то сольёт API-кредиты бесплатными прогонами).
- **Валидация фото:** Claude Vision должна сначала проверить, что это фото живого человека в полный рост — иначе отказ.

## Фазы

### Фаза 0 · UI-демо в квизе (без бэкенда) — параллельный трек
**Статус:** [ ]
- Добавить в `identity-scan-quiz.html` шаг загрузки фото перед финальным экраном.
- Спиннер «ИИ создаёт твоё преображение» с задержкой 2-3 сек.
- На финальном экране заменить 2 референс-фото на: (1) само загруженное фото клиентки, (2) то же фото с overlay «AI-визуализация будет здесь» или с заглушкой.
- **Зачем:** показать UX до того, как тратить $$ на полную реализацию. Можно проверить на 10 живых клиентках, дочитывают ли до конца. Готов прототип для инвесторов.
- **Время:** ~1 час.

### Фаза 1 · Минимальный бэкенд
**Статус:** [ ]
- Поднять Vercel Edge Function `/api/generate-styling`.
- Поднять Supabase проект + бакет `user-photos`.
- Подключить ANTHROPIC_API_KEY и REPLICATE_API_TOKEN в env.
- В демо-квизе заменить заглушку на реальный fetch к API.
- **Тест:** прогнать самой Ксении и 2-3 знакомым. Проверить качество AI-фото.
- **Время:** 2-3 дня.

### Фаза 2 · Готовность к продакшену
**Статус:** [ ]
- Rate limit (1 на email/24ч).
- Капча перед загрузкой (Cloudflare Turnstile или hCaptcha).
- Email-верификация (Resend).
- Логирование: сколько генераций, conversion rate в покупку «Карты стиля».
- Fallback на Midjourney если Nano Banana отказывает.
- **Время:** 2-3 дня.

### Фаза 3 · A/B тест и улучшения
**Статус:** [ ]
- Сравнить конверсию в покупку: «с AI-фото» vs «без AI-фото» (старая версия квиза).
- Если +20% и выше — оставить.
- Если меньше — упростить промпт, попробовать другие модели.

## Итог

**Без бэкенда сейчас:** делаем только Фазу 0 (UI-демо). Это часть текущей сессии.

**С бэкендом:** Фазы 1-3, отдельный спринт на 1-2 недели. Когда Ксения решится — приходить с этим планом.
