# План: A/B-бейкофф моделей генерации образа (preview)

**Дата создания:** 2026-06-16
**Функция:** выбрать модель генерации для preview-якоря квиза — фото клиентки в её Формуле стиля. Закрывает открытый вопрос «какую модель берём» из `2026-05-19-ai-photo-generation.md` (Фаза 1) и `2026-06-09-image-generation-base.md` (Фаза 4, «привязать к API»).
**Стек (memory `ai-stack-and-peer-contact`):** генерация через OpenRouter, кандидаты — Gemini Nano Banana + Seedream 4.5 (мульти-референс) + gpt-image; Flux Pro как fallback. НЕ Midjourney.

## Зачем это отдельный тест

Preview = главный WOW и драйвер покупки. Риск №1 любой генерации лица — **дрейф идентичности** (получается «похожая девушка», не клиентка). Риск №2 — **неверный цвет** (генератор тянет в тёплый беж независимо от подтипа). У нас есть редкое преимущество для объективной оценки: **4 провалидированных эталона** с известными цветотипом/палитрой/фигурой — значит, цветовую и фигурную верность можно мерить не «на глаз», а сверкой с эталоном.

## Что сравниваем

| Модель | Через | Сильная сторона | Что проверяем |
|---|---|---|---|
| Gemini Nano Banana | OpenRouter | скорость, дешевизна | базовый кандидат из плана |
| Seedream 4.5 | OpenRouter | **мульти-референс** (лицо + образ-референс одновременно) | держит ли идентичность лучше за счёт референса |
| gpt-image | OpenRouter | следование сложному текстовому промпту | точность по формуле/подстилю |
| Flux Pro | (резерв) | фотореализм | подключаем, только если топ-3 не дотянули |

## Промпт генерации

Единый для всех моделей — 6-блочная структура из `reference/image-generation/image-generation.md`:
описание лица (из Vision) + одежда из Формулы стиля + сценарий + **палитра подтипа из `12-cvetotipov.md` (named-цвета, без табу)** + editorial-фотостиль + нейтральный фон, мягкий дневной свет.
Для Seedream дополнительно подаём образ-референс (мудборд/комплект подтипа) как второй вход — проверяем выигрыш мульти-референса.

## Рубрика оценки (Ксения, слепо, 1–5 по каждому)

1. **Сохранение идентичности** — это узнаваемо ОНА? (вес ×2 — критичнее всего)
2. **Цветовая верность** — образ в её палитре? (winter_natural → ледяные/чистые, НЕ тёплый беж; сверка с эталоном `12-cvetotipov.md`)
3. **Реализм** — editorial-фото или «пластик/AI-tell»?
4. **Соответствие фигуре** — силуэт по `figure-correction` (hourglass → талия/пояс; rectangle → без футляра)?
5. **Соответствие формуле/подстилю** — Quiet Luxury читается как Quiet Luxury, не «обобщённый лук»?

Практические метрики (не баллы, в таблицу): время/фото, цена/фото, стабильность (доля браков из N прогонов).

## Решающий критерий

Идентичность (п.1) — **gate**: модель с провалом идентичности (<3) выбывает, как бы ни были хороши цвет/реализм. Из прошедших gate — максимум суммы (с весом п.1 ×2). При равенстве — дешевле/быстрее. Мульти-референс Seedream берём, только если он реально поднимает идентичность относительно одно-входных.
