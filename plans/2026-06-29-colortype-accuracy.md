# Точность цветотипа: подтверждение + измерительный пайплайн

**Проблема:** цветотип по одному селфи определяется ненадёжно. Тёплый свет лампы сдвигает баланс белого → «Лето» читается как «Осень». Уверенный, но неверный вердикт бьёт по доверию сильнее всего. Готового надёжного B2B-API нет — топ-приложения (Dressika, BeautyPlus, Manus) это потребительские apps с той же inherent-слабостью по освещению.

**Принцип:** не покупать чужой чёрный ящик (та же ошибка + зависимость + лишняя трансграничная передача 152-ФЗ), а заменить «LLM на глаз» на измерение цвета. Решение фаундера (2026-06-29): **довериться ИИ, точность поднимать измерительным пайплайном** — ручной выбор клиенткой отклонён.

## Фаза 1 — Подтверждение/правка цветотипа `[x]` ОТКАЧЕНО
- Делали: селектор «Знаешь свой цветотип?» на шаге загрузки (override `colortype_known`).
- **Откатили по решению фаундера 2026-06-29:** противоречит обещанию «ИИ определит по фото», большинство клиенток типа не знают → лишний вопрос в момент загрузки. Механизм `colortype_known` в `app/main.py` остался (используется в форме /demo), но из квиза селектор убран.

## Фаза 2 — Измерительный пайплайн (точность + объяснимость) `[ ]`
- Face landmark → пробы пикселей кожи (щека/лоб), волос, радужки.
- Коррекция баланса белого; перевод в Lab.
- Классификация по 3 осям: подтон (тёплый/холодный), светлота, насыщенность → 12 сезонов.
- Выход — не «вайб», а замер: «подтон кожи нейтрально-холодный» (усиливает блок объяснимости).
- Желательно client-side (как Face Type Detector): фото не покидает устройство → снимает вопрос трансграничной передачи.

## Фаза 3 — Качество входа `[ ]`
- Подсказка при съёмке: «дневной свет у окна, нейтральный фон, без фильтров».
- Опционально — лист/карта-референс белого в кадре для калибровки.

## Итог
Фаза 1 откачена (доверяемся ИИ). Точность цветотипа — через Фазу 2 (измерительный пайплайн), приоритет по решению фаундера. Фаза 3 (качество входа) — параллельно, дёшево.
