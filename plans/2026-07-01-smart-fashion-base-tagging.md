# План: методология разметки умной fashion-базы

**Функция:** единый стандарт разметки базы брендов и вещей — не «цена + ссылка», а стиль, силуэт, цветотип, ситуация жизни, капсульность, эффект на фигуру. Цель — движок подбирает по цепочке **бренд → вещь → стиль → фигура → цветотип → ситуация → капсула → рекомендация**.

**Источник материалов:** курс «Алгоритмы имиджа» (`Рабочий стол\АКАДЕМИЯ СТИЛЯ`) — «Коммерческие и исторические стили», «ГИД ПО НАСМОТРЕННОСТИ АИ», «Алгоритм создания капсулы», «База. Тренды / Рациональный гардероб», «Силуэты», «Пропорции», «Анализ конституции тела», «Trendy mama», «Домашний гардероб», «Типы запросов», AI_Stylist_Bonus_v4.

**Главный артефакт:** [`architecture/reference/shopping/fashion-base-tagging.md`](../architecture/reference/shopping/fashion-base-tagging.md) — схема + словари + правила разметки + конвейер (кто заполняет).

**Связь с существующим (не дублируем):**
- `brand-matrix.md` — уровень бренда (бюджет × стилевое поле). Методология его расширяет и задаёт словари.
- `core/catalog.py::Product` — техническая нормализация вещи из фида. Методология говорит, какими смысловыми полями дополнять и откуда их брать.
- План `plans/2026-06-29-affiliate-catalog-base.md` — техника фида/подбора. Этот план — про **смысловую разметку** поверх неё.
