# Кабинет: капсулы (история), сезоны, лист покупок — к ИТМО

**Дата:** 2026-07-02
**Статус:** в работе
**Связан с:** [[plans/2026-07-02-gap-lifecycle-tiers.md]] (Фаза 3 — трекер разрыва в `/me`)

## Зачем
Фаундер решила довести кабинет (сейчас «СКОРО») до рабочего к ИТМО (дедлайн 20 июля) и показать его на
сцене. Кабинет = содержание метода: капсульный гардероб (системность) + разрыв во времени (измеримость).

## Решения фаундера (2026-07-02)
- **Формат демо: живая генерация на сцене.** → Значит генерация ОБЯЗАНА быть устойчивой: пополненный
  ключ + грациозный фолбэк (при 402/таймауте показываем последнюю готовую Карту, не сырую ошибку) +
  параллелизм/скорость. Это требование, а не опция.
- **Срез: полный** — капсула с историей (было → обновление) + переключатель сезонов
  (весна-лето / осень-зима) + живой лист покупок.

## Что уже есть (переиспользуем)
- Капсула: `build_style_card` (base_capsule, capsule_board, looks, styling).
- Лист покупок: `generate_shopping_list` (вещи + бюджет; сейчас тизер в Карте).
- Сезон: генерация умеет весна/лето/осень/зима (`_SEASON_LOOK`, param `season`), но Карта захардкожена
  на `FW 2026-2027`.
- Кабинет `/me` + трекер разрыва (Фаза 3).

### Фаза 5 — Сезоны капсульного гардероба [x] (плюмбинг; контент — после ключа)
- `_CARD_SEASONS` (ss=весна-лето / fw=осень-зима), `build_style_card(diag, season=...)` — убран хардкод
  `FW 2026-2027`, сезон идёт в `gen_req` и в карту (`season`/`season_label`).
- Проброс сезона: анкета Карты (радио-кнопки «Весна–лето / Осень–зима») → `card_build` → воркер;
  текстовый путь `?season=`. Сезон виден в шапке Карты.
- Версии per-season готовы (Фаза 4). Реальный сезонный контент капсулы проверяется после пополнения ключа.

### Фаза 6 — Кабинет: капсула + сезоны + конструктор + лист покупок [x] (база)
- Маршрут `/cabinet` + `CABINET_PAGE`: дашборд по макету фаундера.
- **Конструктор образа** — клиентский: вещи капсулы по слотам, клик/перетаскивание в «Твой образ»
  (по одной вещи на слот), подсветка выбранного, счётчик, «Очистить». Палитра показана как единая
  система цветов («сочетается между собой» = методология выверенной палитры). Проверено кликами в
  Playwright — образ собирается.
- **4 сезона** (весна/лето/осень/зима): вкладки-переключатели, сервер отдаёт капсулу нужного сезона
  из `current_card_by_season`.
- **Лист покупок** — из карты (name + closes_gap).
- Вход из профиля `/me` → «Мой гардероб».
- 59/59 тестов зелёные, дашборд отрендерен и проверен визуально.
- **Осталось (после ключа / отдельно):** фотореалистичная примерка образа «на себе» по кнопке (1
  генерация на выбранную комбинацию); отметки «куплено/ношу» в листе покупок → петля пере-замера;
  карточки истории капсул «было → обновление» (данные Фазы 4 уже есть, нужен UI-блок).

### Фаза 8 — Точность подбора: палитра↔цвет + стоп [x]
- `core/catalog.py`: `_COLOR_FAMILY_KW` + `_color_families()` — имена палитры метода («Чёрная ночь»,
  «Холодный тауп», «Рубиновый») и сырые цвета фида («чёрный», «тауп», «винный») сводятся к одной
  базовой семье. `match_products`: +2 за цвет в палитре, −1.5 за цвет вне палитры, отсечение по
  стоп-семье (горчичный→yellow и т.п.). Раньше подстрока не находила «Чёрная ночь»↔«чёрный».
- Тесты `tests/test_catalog.py` +2. На реальных данных топ-вещи теперь из палитры. 61/61 зелёный.
- **Осталось:** теги стиля (classic/natural/drama/romance) на вещах фида → подбор под подстиль, не
  только цвет; расширить `_FORMULA_CATEGORIES` на 4 стиля.

### Фаза 9 — Packshot-фото (вещь без модели) [ ]
- Запрос фаундера: в конструкторе фото ТОЛЬКО вещи (пиджак = пиджак), без модели и лишних вещей.
- Реальность данных: Ushatava/Lichi — editorial, снимают на моделях (скрейпер берёт `first_image` /
  `photos[0]` = модельный кадр). Packshot — это (а) выбрать предметный кадр из галереи товара, или
  (б) парсить бренды с предметной съёмкой (масс-миддл: LIME, I AM Studio, Choux).
- План: в `scrape_brand.py` добавить эвристику «предметное фото» (кадр на белом/без лица) + при
  расширении каталога брать бренды с packshot-съёмкой. Сетевая работа, проверяется вживую.
