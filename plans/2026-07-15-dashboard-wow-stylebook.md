**Задача (фаундер, 15.07.2026):** улучшить оформление дашборда трекера капсул («сделать вау») и
собрать Style Book новой клиентки по образцу кейса Микуловой — «такой же, а лучше». Всё — в
существующей бренд-системе (Petrogradka: тёплый крем + вино + Cormorant), не менять цвета.

  под текст+свотчи; фото-версия = следующий шаг (см. Фаза 3).

## Фаза 2. «Вау»-дашборд кабинета [x]

`app/main.py::cabinet` + `CABINET_PAGE`. Осталось в бренд-системе, цвета не менялись.
- Дашборд-хиро сверху (сводка одним взглядом, как положено дашборду): кольцо Identity Gap (текущий
  разрыв, анимируется от нуля) + KPI-плитки (Формула · вещей в капсуле · образов из капсулы ·

## Фаза 4. Профиль кабинета + качество фото + правила капсулы везде [x]

- **Профиль-идентичность («стилевая ДНК»)** в шапке кабинета: аватар-инициал, Формула как заголовок,
  чипы Цветотип · Силуэт · Сезон, строка «ты хочешь считываться как …» (want_traits) — наш
  psychology-first ответ на «Style DNA» конкурентов (не «тип гардероба», а желаемое впечатление).
- **Призыв к пере-замеру** при ≥30 днях с последнего замера — гонит петлю трекера (ядро продукта).
- **Связка капсула = конструктор подтверждена:** и кликабельные вещи, и ячейки лука берут один
  `board` из `_visual_capsule`. Клик по вещи капсулы → она встаёт в лук. Не ломали.
- **Качество фото:** `_PHOTO_FINISH` (плёнка/зерно/реальная кожа, без ретуши) + `_LOOK_CANON`
  (актуальный крой, рукав не 3/4, структурная верхняя, без устаревшего) заведены в
  `render_look_on_client` → применяются во ВСЕХ путях рендера (Карта, стилизация, капсула).
  Правила капсулы также в спецификации `image_generation_prompt` (look-generator + стилизация).
- Проверено рендером во всех состояниях, 77 тестов зелёные.

## Фаза 8. Конкурс-пакет: PDF Карты, отзывы, документы, фото-стайлбук [x]

- **PDF Карты стиля идеальный:** `downloadPdf` теперь async — ждёт `document.fonts.ready` и
  ДЕКОДИРОВАНИЕ всех картинок (`img.decode()`) до снимка html2canvas. Это чинит главную беду
  (пустые прямоугольники вместо фото + заголовки в запасном шрифте). Превью-образ тоже guard-нут.
- **Публичные отзывы на лендинге + модерация:** таблица `feedback` +колонка `approved` (миграция);
  `approved_feedback()`, `set_feedback_approved()`; публичный `/api/reviews` (без email);
  в `/metrics` кнопка «показать на сайте» per-отзыв; на лендинге секция «Отзывы клиенток» с JS-fetch
  (скрыта, пока нет одобренных). E2E проверено.
- **Документы конкурса:** `scripts/build_submission_pdf.py` — из `submission/*.md` собирает
  print-ready HTML (A4, типографика, разрывы) в `submission/pdf/`. `markdown`-lib + минимальный
  фолбэк. Открыть → Ctrl/Cmd+P → PDF. Все 7 документов собраны.
- **Фото-стайлбук (платный):** роут `/stylebook` + гейт `_stylebook_access` (админ / premium-флаг /

  генераций. Ссылка в кабинете. Разделы: обложка → где ты сейчас (Gap) → палитра → силуэты →
  капсула(фото) → образы(фото) → итог; кнопка «Скачать PDF» (print).
  **Payment-гейт — заглушка:** реальная оплата ЮKassa не построена (блок G readiness). Сейчас доступ
  даёт админ/флаг; после ЮKassa — привязать к факту оплаты.
