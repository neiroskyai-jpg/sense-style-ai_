# План: Справочник атрибутов стилей для AI-слоя (style-library)

**Дата:** 2026-06-06
**Функция:** Knowledge base атрибутов стилей, питающий промпты генерации образов и анализа гардероба.

## Зачем

У нас есть таксономия (4 стиля + 25 подстилей в `sense-style-method.md`), но в ней нет жёстких атрибутов для AI-генерации: конкретных цветов, принтов, тканей, брендов-референсов и правил сочетаемости. Без этого `look-generator` и `wardrobe-analyzer` будут галлюцинировать образы и нарушать стилевую логику (перестилизация, несочетаемые миксы).

Источник данных — модуль «Основные коммерческие и исторические стили» школы «Алгоритмы имиджа» (17 стилей) + глоссарий. Это та же школа, что Источник 1 нашей методологии, поэтому данные совместимы.

## Фазы

### [x] Фаза 1. Сбор и маппинг
- Извлечь атрибуты 17 стилей из PDF «4 модуль».
- Сопоставить с нашими 25 подстилями (карта соответствия).
- Зафиксировать универсальные правила микса и несочетаемости.

### [x] Фаза 2. Оформление справочника
- Создан `architecture/prompts/style-library.md` v1.0.
- Зарегистрирован в `architecture/prompts/README.md`.

### [ ] Фаза 3. Валидация
- Прогон с Ксенией: соответствуют ли атрибуты её интуитивному пониманию стилей.
- Проверить карту соответствия на пограничных подстилях (quiet luxury, power woman, smart casual).

### [ ] Фаза 4. Интеграция в промпты
- При написании `look-generator.md` — подключить справочник, прогнать hard-rules микса на 5-10 кейсах.
- При написании `wardrobe-analyzer.md` — использовать группы родства для предложений миксов.
