# plans/

Папка для **технических планов реализации функций** Sense Style AI.

## Правила

1. **Один план = одна функция.** Не объединяй несколько функций в один файл и не дроби одну функцию между файлами.
2. **Имя файла:** `YYYY-MM-DD-название-функции.md` (дата создания плана + короткое kebab-case название).
3. **Структура плана:** делится на фазы. У каждой фазы статус — `[ ]` (не сделано) или `[x]` (сделано).
4. **В конце каждого плана** — итоговый блок: реализован целиком или нет, что осталось.
5. **Актуализация — обязательна.** Любой агент, работающий с планом (включая Claude в любой сессии), обязан отмечать сделанные фазы и дописывать, что именно сделано в эту сессию.

Файлы появляются по мере появления задач на функции.
