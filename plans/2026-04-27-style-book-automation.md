# План: автоматизация Style Book (параллельная фоновая работа)

**Дата создания:** 2026-04-27
**Назначение:** строится параллельно с продажами вручную. Каждая ручная сборка = тренировочные данные.
**Дорожная карта замещения:** см. таблицу в `.business/products/overview.md` → MVP-стратегия.

## Цель
К Q4 2026 — 90% Style Book генерируется автоматически по prompt-системе на Claude Sonnet 4.6. Эксперт ревьюит и оставляет персональные нюансы только в Premium-тире.

## Фазы

### Фаза 1 · Q1 (май–июль 2026) · Сбор тренировочных данных + skeleton

- [ ] **1.1** После каждой ручной сборки фиксировать в Notion / отдельном репозитории: вход (квиз-ответы + фото вещей) → выход (готовый PDF) → решения, которые принимал эксперт. Минимум 10 пар к концу Q1.
- [ ] **1.2** Описать в `plans/style-book-prompt-spec.md` структуру финального документа (разделы, длина, стиль) — фиксированный шаблон, под который потом будем строить генерацию.
- [ ] **1.3** Собрать prompt-систему v0.1 для генерации **отдельных блоков** Style Book (описание архетипа, объяснение цвета, текст про Identity Gap). Без сборки целого документа.
- [ ] **1.4** Подключить Claude API (Sonnet 4.6) в Next.js-репо. Тестовый endpoint, который принимает квиз-ответы и возвращает «архетип + 3 предложения объяснения».

### Фаза 2 · Q2 (авг–окт 2026) · 30% AI-генерация

- [ ] **2.1** Расширить prompt-систему до генерации полного раздела «архетип + цветотип + силуэты» — это 30% документа.
- [ ] **2.2** Подключить Claude Vision к анализу фото вещей (выходит структурированный JSON: цвет / тип / материал / стиль).
- [ ] **2.3** Шаблон PDF-генератора (React-PDF или Puppeteer) — собирает финальный документ из блоков.
- [ ] **2.4** Эксперт собирает в Notion → AI-блоки вставляются автоматически → эксперт ревьюит и нажимает «экспорт PDF».

### Фаза 3 · Q3 (ноябрь 2026 – январь 2027) · 70% AI-генерация

- [ ] **3.1** Generate the «капсула из 5 образов» через AI с использованием каталога референсов.
- [ ] **3.2** Шопинг-лист генерируется AI на базе анализа гардероба + Identity Gap.
- [ ] **3.3** Личный кабинет клиентки: вход через Supabase Auth, история Style Book, повторный замер Identity Gap через 2 недели.

### Фаза 4 · Q4 (фев–апр 2027) · 90% AI-генерация

- [ ] **4.1** Light-тир — полностью автоматический. Эксперт включается только если AI выдаёт low-confidence результат.
- [ ] **4.2** Standard — AI собирает целиком, эксперт делает 15-минутный ревью.
- [ ] **4.3** Premium — остаётся с обязательным экспертным ревью (это часть оффера).
- [ ] **4.4** Запуск V2 — клиентское веб-приложение с полным циклом без участия эксперта (для Light/Standard).

## Критическая зависимость от других планов
- Plans/landing — ставит CTA на Tally → Style Book. Из этого плана вытекает потребность в формальном бэкенде (Stripe → Notion / Supabase).
- Plans/quiz-migration — переезд квиза с Tally в Next.js (Q2). До тех пор Tally работает, но без полного контроля над данными.
- Plans/identity-gap-mini-scan — отдельный план на реализацию mini-Scan (7 вопросов «после») и email-триггер через 2 недели. Методология — `.business/products/identity-gap-measurement.md`.

## Риски
- **Качество AI-генерации.** Sonnet 4.6 может не воспроизводить стиль фаундера. Митигация — собирать «золотые примеры» уже в Q1, использовать как few-shot в prompt.
- **Себестоимость токенов.** 1 Style Book целиком = от 30К до 200К токенов (Claude Vision на фото — дорого). Митигация — ограничивать число фото, кэшировать, считать стоимость на каждой фазе.
- **Перфекционизм фаундера.** Главный нетехнический риск. Митигация — выкатывать prompt-систему версиями (v0.1, v0.2, v0.3), не ждать «идеальной».
