# Жизненный цикл Identity Gap: квиз → Карта → подписка

**Дата:** 2026-07-02
**Статус:** в работе

## Зачем

QA-сессия фаундера (тест на себе) вскрыла рассинхрон: экран квиза показал Gap 41%, Карта — 78%.
Разбор показал две вещи:

1. **Баг:** старая закэшированная Карта не инвалидируется при новой диагностике → клиент видит устаревший Gap.
2. **Продуктовое решение фаундера:** Gap — ОДНО честное число на всех уровнях; глубина уходит в РЕШЕНИЕ
   (подстиль, капсула, психотип), а НЕ в новое число. Прежнее «Gap квиза может отличаться от Gap Карты» —
   отвергнуто как fear-sell, противоречащий бренду («психология, не маркетинг»).

Канон — в памяти `gap-single-diagnosis-quiz-to-card.md`.

## Модель (что строим)

Один Gap — один жизненный цикл на трёх уровнях:

| Уровень | Что с Gap | Что добавляет глубина |
|---|---|---|
| **Квиз** (бесплатно) | измерен впервые = **точка отсчёта** (baseline) | — (тизер-воронка) |
| **Карта** (3900₽) | **тот же** Gap, буквально «тот самый разрыв» | психотип → подстиль, палитра, капсула, образы |
| **Подписка** (кабинет) | **тот же** Gap, **закрывается во времени** (64→33→18) | трекер, бот «брать/не брать», пере-замеры |

**Правило целостности:** Gap двигается ТОЛЬКО от реального пере-замера (новые фото текущего состояния =
новая сессия `diagnose()`). Никакого декремента «для красоты». Дельту («−46 п.п.») показываем только при
≥2 реальных замерах; при одном — текущий Gap без дельты.

## Фазы

### Фаза 1 — Фикс кэша Карты (баг рассинхрона) [x]
- `_diag_signature(diag)` — отпечаток по стабильным полям (gap + semantic_field_distribution +
  want_traits), НЕ зависит от refine_substyle (формула/подстиль). Штампуется в Карту как `_diag_sig`.
- `_card_stale(prof)` — Карта устарела, если отпечаток не совпал с текущей диагностикой. Старые Карты
  без отпечатка не трогаем.
- `/card` GET: считает `stale`, показывает старую Карту + плашку «Твоя диагностика обновилась» +
  кнопка «Собрать Карту заново». Ссылка «пересобрать (с анкетой)» → понятное «собрать заново».
- Лимит бесплатных генераций: пересборку при `stale` разрешаем (новый квиз = легитимный повод), в
  `style_card` и `card_build`. Дневная квота `_quota_left()` по-прежнему ограничивает сверху.
- Тесты: `tests/test_style_card.py` +5 (отпечаток/устаревание), все зелёные.
- Файлы: `app/main.py` (import hashlib, `_diag_signature`, `_card_stale`, `build_style_card`,
  `style_card`, `card_build`, шаблон `STYLE_CARD`).
- **Осталось проверить на проде** после пополнения OpenRouter: квиз→Карта дают один Gap; повторный
  квиз с другим Gap → на Карте плашка + пересборка совпадает.

### Фаза 2 — Копирайт «база → глубина» + анкета кнопками [x]
- Экран квиза (`web/identity-scan-quiz.html`): к описанию CTA добавлен «хвост» — «квиз измерил
  *насколько*, Карта достраивает через психотип; **разрыв тот же — маршрут твой личный**». Не обещаем
  новое число, обещаем точность решения.
- Шапка Карты (`STYLE_CARD`): «Identity Gap: N% — **тот самый** разрыв с твоей диагностики. Здесь ты
  видишь, чем именно его закрыть».
- **Анкета Карты — кнопки вместо текста** (запрос фаундера «выбирал кнопками, а не писал»): поля
  `adv`/`balance`/`taboo`/`audience` из `<input type=text>` переведены в чипсы-мультивыбор (макрос
  `chips` + `.chip` CSS в стиле квиза). `_save_deep_intake` читает `getlist` и склеивает — совместимо со
  старым текстом. Визуально проверено скриншотом (Playwright): 4 группы кнопок, ввод текстом убран.
- Тесты: полный прогон 51/51 зелёный. Рендер шаблона без ошибок Jinja.

### Фаза 3 — Трекер разрыва в кабинете + пере-замер [x]
- `tracking.py`: `gap_timeline()` — точки трекера = РАЗНЫЕ замеры; подряд идущие сессии с одним gap
  (эхо квиз+Карта+лид на одном диагнозе) схлопываются в одну точку. `gap_progress()` — дельта
  («−N п.п.») ТОЛЬКО при ≥2 реальных замерах; при одном — точка отсчёта без дельты.
- `/me`: карточка **«Эволюция стилевого разрыва»** — точки с датами (первая = «точка отсчёта»), бары,
  бейдж дельты, честная подпись «двигается только от реального пере-замера», кнопка «Сделать пере-замер»
  (ведёт на квиз = новый замер). Визуально совпало с макетом подписки (проверено скриншотом).
- `_ru_date()` — ISO → «2 июл 2026».
- Тесты `tests/test_tracking.py` +4: схлопывание эха, нет дельты при 1 замере, дельта при 2, None без истории.
- **Осознанно вне scope (отдельные фичи, не про Gap):** сам платный кабинет по подписке (сейчас «СКОРО»),
  бот «брать/не брать» в Telegram, автоматический вызов пере-замера по фото — когда будет готов UI подписки.
  Пере-замер сейчас = повторный квиз, который и так пишет новую точку.
