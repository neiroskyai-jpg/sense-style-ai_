# План: заявка на Junior ML Contest ИТМО (AI Talent Hub)

**Категория:** ИИ-стартап (AI-First бизнес-проект).
**Целевая волна:** 3-я — подача 3–20 июля 2026, защита 27–29 июля, итоги 31 июля.
**Приз:** бюджетное место в онлайн-магистратуре AI Talent Hub («Искусственный интеллект» / «Управление ИИ-продуктами»).
**Участник:** founder как IT-специалист с опытом в AI/ML; в заявке описывается личный вклад в AI-слой.

## Контекст и вводные

- Допуск подтверждён по пути «IT-специалист с опытом работы в AI/ML».
- Магистратура — реальная цель (мотивационное письмо честное).
- Технически: есть наработки кода, нет публичного репо, MVP, метрик и оформленной обратной связи.
- Формат защиты: 5 мин презентация + 7 мин вопросы.

## Критерии оценки и наша позиция

| Критерий | Позиция | Стратегия |
|---|---|---|
| Разработка и инженерия (Git/Docker/CI/MLOps) | слабо | публичный репо + Docker + базовый CI + чистый пайплайн |
| Data Science (EDA, выбор моделей, эксперименты) | слабо | переупаковка: методология = размеченные данные; eval Vision vs экспертный ground truth; A/B генераторов |
| Применение ИИ (мультимодальность, агенты, RAG) | сильно | Vision-анализ + генерация + агентный промпт-пайплайн + RAG по методологии |
| Продуктовое мышление (проблема, ЦА, MVP, импакт) | очень сильно | Identity Gap, ЦА, конкуренты, импакт в % до/после |
| Мотивация | сильно | искреннее желание учиться в AITH |

## Ключевая стратегия по Data Science

Мы не обучаем свои модели — оркеструем Claude Vision + генерацию. Чтобы не провалить DS:
- **Данные:** 4 стиля, 25 подстилей, цветотипы, табу-цвета, Identity Gap в % — структурированная разметка.
- **Ground truth:** дипломный стайл-бук Ксении с экспертными правками + кейсы из `architecture/cases/`.
- **Эксперимент:** accuracy Vision-диагностики vs экспертная разметка (типаж/фигура/цветотип); доля попаданий генерации в табу-цвета; A/B генераторов (Gemini Nano Banana vs Seedream 4.5).
- **Импакт:** Identity Gap «до/после» в %.

## Обязательные артефакты для подачи

1. Репозиторий (GitHub, публичный) — обязателен.
2. MVP (работающий, хотя бы 1 сквозной сценарий).
3. Метрики использования/эффективности.
4. Обратная связь пользователей (3–5 отзывов + кейсы).
5. Описание проекта (PDF, до 3 страниц: задача → данные → методы → результаты).
6. Презентация (PDF, по шаблону ИТМО).
7. CV (PDF) + мотивационное письмо (PDF).

## Фазы
