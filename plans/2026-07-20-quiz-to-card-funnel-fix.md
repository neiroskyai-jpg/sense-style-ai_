# Починка воронки квиз → Карта (живой прогон перед конкурсом)

**Дата:** 2026-07-20
**Контекст:** финальная проверка перед подачей на конкурс ИТМО. Тесты были зелёные, но живой
прогон в браузере вскрыл обрыв воронки, который тесты не покрывали.

## Проблема

Клиентка проходит все 14 вопросов квиза, жмёт «Пропустить · показать результат без фото»,
видит свой результат — и на кнопке «Получить полную Карту стиля» попадает на экран
«Сначала — диагностика». Квиз пройден впустую.

Причина: `job_id` заводился только в `/api/analyze`, то есть только вместе с фото. Без фото
диагноз нигде на сервере не сохранялся, и `/card` не находил его ни по `?from_job=`, ни по сессии.

Это самый вероятный путь для жюри: чужой человек не станет грузить своё фото.

## Фазы

- [x] **Фаза 1. Диагностика без фото доезжает до сервера.**
  Эндпоинт `POST /api/quiz-diagnosis` (`app/main.py`): принимает ответы квиза, считает диагноз
  через `diagnose()`, кладёт под `job_id` (в `_JOBS`, на диск и в `session["last_job"]`).
  В квизе — `ensureQuizDiagnosis()` на экране результата плюс перехват клика по CTA, если ответ
  сервера ещё не пришёл.

- [x] **Фаза 2. Фолбэк на случай недоступного провайдера.**
  `_quiz_only_diag()` собирает честный диагноз из одних ответов квиза, когда `diagnose()` падает
  (кончился ключ, лимит, сеть). Воронка важнее полноты диагноза: лучше отдать формулу и Gap
  из квиза, чем развернуть клиентку.

- [x] **Фаза 3. Капсула перестала противоречить формуле.**
  Фолбэк клал в `base_style` русское «Классика», а `_FORMULA_CATEGORIES` в `core/catalog.py`
  ждёт код (`classic`). Словарь молча возвращал пустоту, фильтр по формуле отключался — в капсулу
  «Классики» падали кружевная накидка и юбка с воланами. Теперь код + `semantic_field_distribution`
  с доминантой. Проверено живьём: пиджак, сорочка, брюки со стрелками.

- [x] **Фаза 4. Индекс — одно число на всех экранах.**
  Карта показывала `100 − gap` (69%), кабинет — сырой `gap` (31%) под той же подписью «Индекс».
  Кабинет приведён к формуле Карты.

- [x] **Фаза 5. Вёрстка и мелочи.**
  `break-inside: avoid` для списка «Преображения» в `web/index.html` — пункт рвался между
  колонками, хвост «Massimo Dutti» уезжал вправо отдельной строкой без галочки.
  Добавлен `/favicon.ico` (SVG-монограмма) — был 404 на каждой странице.

- [x] **Фаза 6. Тесты.**
  `tests/test_quiz_without_photo.py` — 9 тестов на всё вышеперечисленное, включая поведение
  при лежащем провайдере. Полный прогон: **251 passed**.

## Итог

Реализован целиком. Воронка квиз → Карта → кабинет проверена вживую в браузере и доходит до конца
без фото и без работающей генерации.
