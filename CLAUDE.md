# Sense Style AI — навигация по проекту

Этот документ — карта проекта. Читай его в начале каждой сессии, чтобы понимать, где что лежит и как с этим работать.

## Что это за проект

**Sense Style AI** — персональный AI-стилист с Vision-анализом для женщин 30–50 в точках перехода (карьера, материнство, эмиграция, статус). Анализирует одежду через психологический профиль клиентки, а не абстрактно. Методология — psychology-first, опирается на Self-Discrepancy Theory (Higgins, 1987) и Enclothed Cognition (Adam & Galinsky, 2012).

**Источники правды:**
- **По продукту:** [`architecture/2026-04-27-mvp-spec.md`](architecture/2026-04-27-mvp-spec.md)
- **По методу (как работает диагностика):** [`architecture/sense-style-method.md`](architecture/sense-style-method.md)
- **По визуалу:** [`architecture/visual-direction.md`](architecture/visual-direction.md)

Бизнес-контекст и стратегия — в `.business/INDEX.md`.

## Ключевые принципы проекта

1. **Научный подход (психология прежде моды).** Все продуктовые и контентные решения опираются на Self-Discrepancy Theory и Enclothed Cognition. Никакой эзотерики, никакой «интуиции стилиста».
2. **Системность (AI как архитектор).** Гардероб = база данных и алгоритм. AI — строгий инструмент построения системы, а не «развлекательная фича для миксов».
3. **Измеримая трансформация.** Продаём переход «неприметность → уверенность». Identity Gap у клиентки замеряется в % «до/после» — без замера нет результата.

## Стек

- **Frontend:** Next.js 14 (App Router) + TypeScript + Tailwind CSS + Framer Motion.
- **База данных:** Supabase (PostgreSQL + Auth magic-link + Storage + Edge Functions).
- **AI:** Claude Vision API для анализа фото вещей, `claude-sonnet-4-6` для генерации капсул и шопинг-листов. Один запрос = фото + полный Gap-профиль клиентки.
- **Платежи:** ЮKassa — основной эквайринг для РФ. Stripe — для зарубежных клиенток через нерезидентское юрлицо или агрегатор.
- **Email:** UniSender transactional API (вход по magic-link) — `core/auth.py`, нужны env `UNISENDER_API_KEY` + `UNISENDER_FROM_EMAIL`; без них — dev-фолбэк (ссылка в лог). Маркетинговая рассылка НЕ автоматизирована: согласие + выгрузка `/metrics/unisender.csv` → серию настраивать в UniSender вручную.
- **Аналитика:** GA4 + GTM + Meta Pixel.
- **Погода:** OpenWeatherMap API (для Daily Style в P2).
- **Хостинг:** Vercel (free tier до ~100К запросов/мес).
- **Квиз:** HTML-прототип `Identity Scan` лежит **у фаундера, не в репо**. Перед началом фронта на Next.js — добавить в `.business/assets/identity-scan-quiz.html`.

## Tone of Voice для AI-генерации

Claude генерирует тексты Style Book, описаний образов, email-серий и постов. Эти тексты должны звучать **как Ксения Колупаева, не как ChatGPT**.

**Правила:**
- Тон — тёплый, но **не сюсюкающий**.
- Обращение — на **«ты»**.
- Объяснения — **через психологию**, а не через моду («твой образ работает на закрытие Identity Gap», а не «это сейчас тренд»).
- **Без восклицательных знаков через слово.** Максимум один восклицательный на длинный абзац, и только если он несёт смысл.
- **Запрещены:** «потрясающе!», «великолепно!», «вау», «изумительно», «обалденно», «вы молодец», смайлики, эмодзи в теле текста.
- Запрещены пустые усилители без смысла («просто супер», «настоящая магия»).
- Длинные предложения на 3 уровня вложенности — нет. Короткие, ритмичные. Иногда — длинные, когда нужно объяснить мысль через нюанс.
- Допустимая лексика: «работает», «закрывает», «тебе подходит», «обрати внимание», «вот почему», «здесь важно понять», «попробуй».

**Пример правильно:**
> Тёмно-зелёный жакет работает на твой запрос «выглядеть властно». Сатурация цвета даёт авторитет, прямой силуэт убирает «девочковость», в которой ты застряла. Надевай на встречи, где нужно, чтобы тебя слышали.

**Пример неправильно (так — нельзя):**
> Этот восхитительный тёмно-зелёный жакет — просто настоящая находка для вас! 🔥 Он сделает вас невероятно властной и стильной! Обязательно носите его на важные встречи, и все будут вами восхищаться! ✨

При генерации **любого** текста Claude пропускает результат через эти 8 правил. Если текст звучит как универсальный «контент-маркетолог» — переписать.

## Папка `plans/`

Технические планы реализации **отдельных функций**. Здесь живёт информация **как делаем конкретную функцию**. Один план = одна функция. Архитектурные решения, которые шире одной функции, идут в `architecture/`.

## Как работать с проектом

- Перед любой задачей — прочитай `CLAUDE.md`, затем `architecture/2026-04-27-mvp-spec.md` (если задача продуктовая или техническая), затем нужные документы из `.business/` и `plans/`.
- Если меняется продукт или архитектура — обнови `architecture/2026-04-27-mvp-spec.md` и приведи `.business/` под него.
- Если меняется бизнес-логика — обнови `.business/INDEX.md` и нужные файлы.
- Если меняется технический план функции — обнови соответствующий план в `plans/`.
- Если появляется новая важная папка или документ — обнови этот `CLAUDE.md`.

## ВАЖНО: план для каждой новой функции

Любая функция, которую мы создаём в любом чате, всегда оформляется планом в папке `plans/`.

Правила:

1. Один план = одна функция. Если план уже есть — работаем с ним.
2. Имя файла: `YYYY-MM-DD-название-функции.md`.
3. План делится на фазы. У каждой фазы статус `[ ]` или `[x]`.
4. В конце плана — итоговый блок: реализован целиком или нет, что осталось.
5. Любой агент обязан актуализировать план после каждой сессии.

## ВАЖНО: завершение каждого чата

В конце каждой сессии записывай рефлексию в файл `.business/история/YYYY-MM-DD-краткое-название.md` (создавай папку при необходимости).

Формат:

1. Какая задача была поставлена.
2. Как я её решал.
3. Решил ли — да / нет / частично.
4. Эффективно ли решение, что можно было лучше.
5. Как было и как стало.
