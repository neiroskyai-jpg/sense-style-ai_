# Архитектура: база знаний + Vision-квиз + шопинг

**Дата:** 2026-06-09
**Статус:** принятая архитектура (решения фаундера: тренды — гибрид; шопинг — гибрид со стартом на брендах).
**Связано:** `reference/KNOWLEDGE-BASE.md` (карта покрытия), `2026-04-27-mvp-spec.md` (продукт), промпты `prompts/`.

Документ отвечает на два вопроса фаундера: (1) как разложить знания, чтобы они оставались актуальными и я их брал → анализировал → применял; (2) как устроен Vision-квиз от бесплатного фото до образов и реальных вещей в магазине.

---

## 1. Два слоя знаний (по «скорости порчи»)

| Слой | Что | Темп изменений | Где |
|---|---|---|---|
| **Эвергрин (метод)** | цвет, фигура, принты, стилевые типажи, психология, глоссарий | редко, версионируется (v1.1…) | `reference/colortypes`, `figure-correction`, `print-mixing`, `glossary`, `prototypes`, `scenarios` |
| **Актуальное (тренды)** | тренд-аналитика, новости моды, стили 2026-2027, новые прототипы | по сезонам, со сроком годности | `reference/trends/` |

Правило связи: **тренд никогда не перебивает эвергрин.** Модное палаццо не отменяет табу-цвет цветотипа и не ломает силуэт фигуры. При сборке образа эвергрин — это фильтр, тренд — это акцент внутри фильтра.

## 2. Цикл «беру → анализирую → применяю»

- **Беру.** Источник падает в `raw-inbox/` (PDF, скрин) или `raw-inbox/transcripts/` (расшифровка встречи). Гибридно: твои материалы — основа; мой веб-поиск новостей моды — дополнение, всё с твоим подтверждением перед выкаткой.
- **Анализирую.** Каждый тренд раскладываю не как «модно X», а с привязкой: **кому по фигуре · в какой палитре · к какому полю впечатления · до какого срока**. Без привязки тренд в базу не идёт.
- **Применяю.** `look-generator` берёт 1-2 актуальных акцента из `trends/`, прогоняет через цвет и фигуру. Устаревшее уезжает в `trends/archive/`.

## 3. Структура `reference/trends/`

```
trends/
  README.md                ← как обновляется, гибридный процесс
  YYYY-FW.md / YYYY-SS.md   ← сезонные тренды, поле «действует до»
  styles-2026-2027.md       ← макротренды периода
  prototypes-news.md        ← пополнение библиотеки прототипов (новые фигуры-векторы)
  fashion-news.md           ← датированная лента находок
  archive/                  ← вышедшее из срока
```

Формат записи тренда (обязательные поля): **тренд · сезон/срок · кому по фигуре · в какой палитре/полю впечатления · как применить · источник**. `prototypes-news.md` дополняет `prototypes/prototype-library.md` (каждый прототип — со статусом ✅ после валидации Ксенией).

## 4. Vision-квиз: сквозная цепочка

```
1. Фото (бесплатно, в квизе — предпоследний шаг, см. mvp-spec)
2. VISION-АНАЛИЗ (Claude Vision):
   • цветотип — драпировка по фото (подтон/глубина/контраст) → 1 из 12
   • фигура — по фото (+замеры если есть) → 1 из 5, с оговоркой «фото в облегающем точнее»
   • контраст внешности — для подбора прототипа
3. + ответы квиза → formula-diagnostic → Формула стиля + Identity Gap %
4. look-generator → капсула/образы (цвет + фигура + принт + тренд-акцент + сценарий)
5. SHOPPING-СЛОЙ → реальные вещи под образы (фильтр: палитра + фигура + бюджет)
6. PREVIEW → 2 образа с лицом/фигурой клиентки (WOW, бесплатно)
```

Vision — не «определи цвет», а полный мост: с одного фото снимаем цветотип + фигуру + контраст, и это сразу едет в готовый движок из 6 баз. Промпт `prompts/vision-analyzer.md` (✅ v1.0, 2026-06-12) превращает фото в поля, которые ждёт `formula-diagnostic`; методы чтения — `reference/colortypes/photo-reading.md` и `reference/figure-correction/body-reading.md`.

**Стек и операционка (решение 2026-06-09, см. `reference/image-generation/image-generation.md` §5a):**
- Анализ фото — **Claude Vision** (наш стек). Генерация preview — **Gemini 2.5 Flash Image (Nano Banana)**, image-to-image сохраняет лицо, ~1-3 ₽/изобр. Midjourney — НЕ продакшен (учебный инструмент). Провайдеры можно держать под одним API через OpenRouter.
- **Сжимать фото на входе** перед Vision (большие изображения — медленно/дорого).
- **Генерировать preview асинхронно** — анализ/формулу показываем сразу, картинки догружаются.
- **Каждый компонент тестировать отдельно, потом склеивать** (наш текущий подход — базы/промпты собираются и валидируются по одному).

## 5. Shopping-слой (решение: гибрид)

Подбор реальных вещей под собранные образы.

- **Фаза 1 — MVP, кураторские бренд-листы.** Бренд-листы по сегментам бюджета уже есть в курсе. AI выдаёт «вещь + бренд + поисковый запрос», отфильтрованные по палитре/фигуре/бюджету. Ноль интеграций, старт сразу. Минус — без прямых ссылок/наличия.
- **Фаза 2 — апгрейд, живой каталог.** Фиды Lamoda/Wildberries/Ozon → реальные SKU со ссылкой, ценой, наличием; affiliate-монетизация. Добавляем после первых продаж, для топ-магазинов, когда понятно, что конвертит.

Почему так: фаза 1 доказывает воронку дёшево (lean MVP, Vercel free tier), фаза 2 — это монетизация и точность, а не блокер запуска. Артефакты готовы: `reference/shopping/brand-matrix.md` (✅ v0.1 черновик — РФ-бренды × сегмент бюджета × стилевое поле + «5 групп товаров») и промпт `prompts/shopping-list.md` (✅ v1.0). Уточнение к фазе 1: «ассортиментная матрица» из курса — это методология структуры капсулы (5 групп товаров), а НЕ список брендов; бренды собраны веб-поиском и требуют валидации Ксенией. Бренд привязан к бюджету и стилевому полю, цвет/силуэт фильтруются на уровне вещи.

## 6. Free / Paid (по mvp-spec)

- **Бесплатно (квиз):** Vision-анализ (цвет/фигура/формула + Identity Gap %) + 2 preview-образа + 2-3 shoppable-вещи как тизер.
- **Платно (тарифы):** полная капсула + полный шопинг-лист + сценарные образы. Без изменений к тарифной сетке.

## 8. Порядок реализации (предложение)

1. Завести `trends/` + `shopping/` скелеты (структура и формат записи).
2. Добить эвергрин-очередь из `KNOWLEDGE-BASE.md` (стилевые типажи — ядро).
3. `vision-analyzer.md` — мост фото → движок (нужен для квиза).
4. `shopping-list.md` фаза 1 (бренд-матрица из курса).
5. Фронт квиза подключает цепочку.
6. Фаза 2 шопинга (живой каталог) — после первых продаж.
