# Data residency и комплаенс 152-ФЗ

Решение по хранению и обработке персональных данных (ПД) клиенток. Не юр. заключение —
финальные формулировки через юриста (`.business/legal/`).

## Требования закона (152-ФЗ)

- **Локализация (ст. 18 ч. 5):** запись/хранение/обновление ПД граждан РФ — в базах **на территории РФ**.
- **Трансграничная передача (ст. 12):** передача за рубеж (любой иностранный AI) — после локализации,
  с **согласием** субъекта и **уведомлением Роскомнадзора** (с 2022 — до начала передачи).
- **Фото:** используется для подбора стиля, НЕ для установления личности → вероятно «обычные» ПД,
  а не биометрия (биометрия = для идентификации). **Требует подтверждения юристом.**

## Принятая архитектура (вариант «РФ-ядро + Gemini трансгранично с согласием»)

| Слой | Решение |
|---|---|
| Хостинг + БД | **Amvera Cloud (РФ, amvera.ru)** — серверы и БД в России (локализация). |
| Хранение | В РФ-БД: аккаунт, результаты, история Identity Gap (трекинг). |
| Фото | Эфемерно: принимаем → отправляем в генерацию → **не храним после обработки**. |
| Текст / Vision / Генерация | **Gemini (Google, США)** ради качества → трансгранично, **с явным согласием**. |
| Согласие | Перед загрузкой фото — два согласия: на обработку ПД и на трансграничную передачу в AI (Google, США). Без них обработка не запускается (реализовано в `app/main.py`). |
| Уведомление РКН | До запуска коммерческой версии — уведомление о трансграничной передаче. |

Почему так: Gemini заметно сильнее РФ-моделей в identity-preserving рендере (наша ключевая
фича). РФ-локализация + согласие + уведомление РКН закрывают закон, не жертвуя качеством.
Альтернативы на будущее: РФ-модели (GigaChat / Kandinsky) или self-host (Flux + InsightFace)
на РФ-GPU — если потребуется убрать трансграничную передачу полностью. Провайдер моделей
сменяем за конфиг (`core/provider.py`), переход не ломает пайплайн.

## Реализовано в коде

- Слой согласия (чекбоксы + серверная проверка) — `app/main.py` (`_consent_ok`).
- Путь БД через `SENSE_DATA_DIR` → persistent-том Amvera (`/data`).
- Дневная квота вызовов (защита ключа) — `core/tracking.py`.
- Деплой: `Dockerfile` (web), `amvera.yml`.

## Осталось (юрист + production)

- Политика конфиденциальности, формы согласия, оферта — `.business/legal/`.
- Классификация фото (биометрия / обычные ПД) — юрист.
- Уведомление РКН (оператор ПД + трансграничная передача).
- Демо для конкурса — на тестовых/своих фото с согласием; Amvera + РКН — production-роадмап.

## Чек-лист оператора по 152-ФЗ (источник: tilda.education/articles-personal-data-law)

| Требование | Статус | Примечание |
|---|---|---|
| Локализация ПД в РФ (БД) | ✅ (план) | Amvera (РФ) + `SENSE_DATA_DIR=/data` |
| Согласие под каждой формой сбора | ✅ тех. | чекбоксы + проверка на сервере; текст согласия — юрист |
| Политика конфиденциальности (отд. страница + ссылка в подвале) | ⚠️ | нужна страница `/privacy` + ссылка в футере на сайте. Штраф 30–60к |
| Cookie-баннер + согласие | ⚠️ | добавить баннер на Vercel-сайт |
| Уведомление РКН (реестр операторов, форма №180 от 28.10.2022) | ⚠️ юрист | подать до старта |
| **Трансграничная передача: уведомление РКН + ожидание 10 раб. дней ДО передачи** | ⚠️ блокер prod | США — «неадекватная защита» → строгий режим. **Реальные фото в Gemini нельзя до подачи и ожидания.** Для демо — тестовые/свои фото |
| Ответ на запросы субъектов за 10 раб. дней | ⚠️ процесс | регламент + канал обращения. Штраф 40–80к |
| Уведомление РКН об утечке за 24 ч | ⚠️ процесс | план реагирования |
| Внутренние документы, ответственный за ПД, антивирус/файрвол | ⚠️ юрист/прод | для юрлица |

**Штрафы (мотивация делать правильно):** обработка без согласия — 300–700к (повтор 1–1,5 млн);
утечка — до 15 млн; нет политики — 30–60к. Поэтому не переобещаем и оформляем по закону.
