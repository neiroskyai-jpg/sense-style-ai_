# architecture/

Сквозные системные документы проекта, которые **переживают отдельные планы**.

## Что сюда кладём
- Мастер-спецификация продукта (что строим, какой стек, какая архитектура).
- Схема базы данных.
- Промпт-библиотека для AI-слоя.
- Архитектурные решения, которые касаются нескольких функций сразу.

## Чем `architecture/` отличается от `plans/` и `.business/`

| Папка | Что внутри | Срок жизни |
|---|---|---|
| `.business/` | **Зачем** делаем (стратегия, аудитория, цели, экономика) | Долгий. Корректируется при смене стратегии. |
| `architecture/` | **Что и как** строим как систему (стек, БД, промпты, мастер-спека) | Долгий. Корректируется при смене архитектуры. |
| `plans/` | **Конкретная функция, которую делаем сейчас** (один план = одна функция) | Короткий. Архивируется после реализации в `plans/archive/`. |

## Источник правды
**`architecture/2026-04-27-mvp-spec.md` — единственный источник правды по продукту.**
Все противоречия между `.business/` и спекой — переписываем в пользу спеки.

## Конвенция именования
Файлы документов: `YYYY-MM-DD-название-в-kebab-case.md`.
Для безымянных «вечных» документов (например, схема БД) — `<тема>.md` без даты.
