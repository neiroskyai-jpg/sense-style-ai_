# Лексикон впечатления — характеристики анкеты с привязкой к полям

**Тип:** knowledge base для AI-слоя. Дословный список прилагательных-маркеров впечатления (вопросы 4-5 анкеты «Алгоритмы имиджа») с привязкой к 4 семантическим полям. Без этого списка `formula-diagnostic` не может посчитать распределение по полям → Формулу стиля и Identity Gap.

**Источник:** анкета школы «Алгоритмы имиджа», вопросы 4 («какое впечатление производите сейчас») и 5 («какое хотели бы после изменения»). Оба вопроса используют ОДИН список опций.

**Статус:** v1.0, **требует сверки с оригиналом анкеты** — выписано с фото, возможны неточности в 1-2 словах. Считаю **66** слов; метод в заголовке говорит «80», но в скобках по полям даёт 18/17/16/15 = 66. Свериться с Ксенией: реальное число и полный список.

**Жёсткое правило:** список — калиброванный инструмент. Слова НЕ добавляем «вне системы» — каждое привязано к полю, баланс полей держит честность замера. Расширять можно только осознанно (новое слово → явно в поле, с сохранением баланса, с валидацией Ксенией).

## Как пользуется AI

`formula-diagnostic` ШАГ 2: выбранные клиенткой слова (Q5 = желаемое, Q4 = текущее) раскладывает по этим 4 полям → распределение в % → базовый стиль + Формула + Identity Gap (field-aware, Q4 vs Q5). Это тот же список, по которому считается `now_field_distribution` и `semantic_field_distribution`.

## Заметки по UX квиза (не «больше слов», а удобнее выбор)

Анкета школы — бумажная, под живого стилиста. В самостоятельном квизе риск не «мало слов», а **перегруз и отвал**. Поэтому:
- Оставляем тот же список (не добавляем) — но **группируем визуально по полям/темам** для лёгкого сканирования.
- Абстрактным словам (импозантная, несравненная, притязательная) — короткая подсказка простым языком.
- Обязательный под-шаг «выбери **3 главные**» из желаемых (метод требует 3 главные для Формулы).
- Выбор «10 и более» оставляем как в методике.

> При расхождении с оригиналом анкеты / подходом Ксении — выигрывает оригинал; список правится.
