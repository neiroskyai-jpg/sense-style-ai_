# Справочник прототипов — база подбора визуального вектора

**Тип:** knowledge base для AI-слоя. Подключается к `formula-diagnostic` (ШАГ 5 «Подбор прототипа»).

**Источник метода:** 7-шаговый алгоритм `sense-style-method.md` + дипломная методика Ксении Колупаевой, подтверждённая директором по стилю школы «Алгоритмы имиджа» (см. [[director-style-validation]] / `.business/assets/sense-style-site/photos/client-case/`).

**Статус:** v1.0. Метод подбора — стабилен. Сама библиотека прототипов — **растущая, наполняется и валидируется Ксенией**. ИИ не выдумывает прототип «из головы»: сначала ищет в этой библиотеке, при отсутствии — предлагает кандидата и помечает «требует подтверждения Ксенией».

**Зачем база:** прототип — самое уязвимое место для ИИ (выдаёт вероятную, но случайную знаменитость). Директор прямо предупреждает: рекомендации ИИ по типажу/прототипу сверяем с реальными знаниями. Поэтому прототип берём из выверенного списка, а не генерируем.

---

## Что такое прототип

Прототип — публичная фигура, чей образ читается так, как клиентка **хочет** читаться. Это **вектор, а не шаблон**: мы не копируем гардероб, а берём из него только образы, комплиментарные цветотипу и фигуре клиентки.

## Правило подбора (двойная проверка + фильтр)

Прототип подходит, только если совпадает по **обеим** осям:

1. **Ось впечатления.** 3 главные прилагательные клиентки (желаемое впечатление, поле Q5) = «считывание» прототипа. Прототип воспринимается так, как клиентка хочет.
2. **Ось физики.** Тип фигуры (или близкий) + рост/пропорции + **контраст и глубина внешности** (high/medium/low, осветлённый/затемнённый) + возрастной код.

Затем **фильтр по цветотипу:**
- Цветотип прототипа **не обязан** совпадать с цветотипом клиентки.
- Из образов прототипа берём только **комплиментарные палитре клиентки**, остальные перекрашиваем в её палитру. Дублировать 100% образов нельзя.
- **Контраст и глубина важнее сезона.** Прототип другого сезона годится, если совпадает уровень контраста и затемнённости (пример: Бекхем — натуральная осень, тёплая; Ксения — натуральная зима, холодная; но оба среднеконтрастные, затемнённые, мало серого пигмента → прототип читается верно, образы фильтруем по холодной палитре).

> Нельзя: выбирать прототип по ассоциации внешности («похожа лицом»). Только по впечатлению + физике.

## Как ИИ пользуется

1. `formula-diagnostic` ШАГ 5: по доминантному полю впечатления (Драма / Классика / Романтика / Натуральный) + типу фигуры + контрасту берёт кандидата из таблицы ниже.
2. Применяет фильтр по цветотипу (раздел выше) — отбирает только комплиментарные образы прототипа.
3. Прототип уходит в `visual_formula` как референс-вектор и в мудборд стайл-бука.
4. Если в библиотеке нет подходящего — ИИ предлагает кандидата и явно помечает «требует подтверждения Ксенией», прежде чем строить на нём стайл-бук.

## Библиотека прототипов

Колонки: поле впечатления · фигура · контраст/глубина · сезон-ориентир · чем читается. Сезон прототипа — справочно (образы всё равно фильтруются под клиентку).

Источник — финальный метод Ксении (`.business/products/метод психологии моды/sense-style-method.docx`, раздел «25 подстилей» и «5 типов фигур»). Прототипы подобраны автором метода → статус ✅ (валидированы Ксенией). Эталон — Виктория Бекхем (подтверждён директором на дипломе).
