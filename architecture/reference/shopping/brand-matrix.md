# Бренд-матрица — РФ-бренды × бюджет × стилевое поле

База для shopping-слоя (фаза 1): какие бренды рекомендовать под формулу и бюджет клиентки. Потребитель — промпт `prompts/shopping-list.md`: по `price_segment` + полям Формулы стиля выбирает бренды, дальше предлагает «вещь + бренд + поисковый запрос».

> **НЕ ЭВЕРГРИН.** Бренды приходят/уходят, наценки и наличие меняются. Это слой со сроком годности: ревизовать раз в сезон-полугодие, версионировать. Эвергрин-метод (цвет/фигура/стиль) bren-matrix не перебивает — фильтр палитры/силуэта идёт на уровне конкретной вещи в `shopping-list`, не на уровне бренда.

**Статус:** v0.1 ЧЕРНОВИК (2026-06-12), собран веб-поиском + знанием рынка. **Требует валидации Ксенией:** состав брендов, сегментация по цене, привязка к стилевым полям, наличие в её регионе. Источники: [vc.ru ТОП-35](https://vc.ru/id3748947/2811325-30-luchshikh-brendov-zhenskoy-odezhdy), [kp.ru](https://www.kp.ru/woman/moda/luchshie-rossijskie-brendy-odezhdy/), [t-j.ru обувь](https://t-j.ru/list/russian-shoes/), [lenta.ru сумки](https://lenta.ru/articles/2026/01/27/luchshie-rossiyskie-brendy-sumok/).

## Принцип привязки

Бренд НЕ привязан к палитре (любой бренд делает вещи всех цветов — цвет фильтруется на уровне SKU). Бренд привязан к:
1. **Сегменту бюджета** — совпадает с `price_segment` клиентки: `low | middle | high | luxury`.
2. **Стилевому полю(ям) метода** — `classic | drama | romance | natural` (стилевая ДНК бренда). Бренд может покрывать 2 поля.
3. **Сильным категориям** — где у бренда лучшая экспертиза (костюмы / трикотаж / платья / верхняя / обувь / сумки).

Сегменты ↔ цены (ориентир, женская одежда):

| price_segment | Сегмент | Жакет | Платье | Пальто |
|---|---|---|---|---|
| `low` | масс-маркет | 4–10к | 3–8к | 8–15к |
| `middle` | средний | 15–30к | 12–25к | 25–50к |
| `high` | доступный премиум | 40–80к | 30–100к | 60–150к |
| `luxury` | премиум-люкс | 80к+ | 100к+ | 150к+ |

## Методология структуры капсулы — «5 групп товаров»

Из курса «Создание ассортиментной матрицы». Логика ритейл-матрицы, применённая к капсуле клиентки: классификация вещей по **роли в системе** (дополняет правило связности в `look-generator`). Используется shopping-list для приоритета покупок.

| Группа | Что это в капсуле | Приоритет покупки |
|---|---|---|
| **Локомотивы** | база, носится постоянно (футболки, базовый верх/низ) | 1 — фундамент |
| **Комплексные** | готовое решение целиком (костюм: даёт образ + раскладывается на верх/низ) | 1 — максимум комбинаций за вещь |
| **Сопутствующие** | дополняют базу (доп. низы — юбки, брюки) | 2 — расширяют комбинаторику |
| **Статусные** | аксессуары, делающие образ (сумка, украшения, ремень, палантин) | 2 — апгрейд впечатления, «дорого» |
| **Заменители** | альтернативы внутри роли (вместо одной юбки — другая) | 3 — по мере надобности |

**6 шагов сборки капсулы (шпаргалка курса):**
1. Выбрать базу (костюм — комплексный товар).
2. Ориентир на цвет + добавить дополнительные низы (юбки, брюки).
3. Дополнить платьями.
4. Подобрать верх (на 1 низ ≈ 2 верха).
5. Добавить аксессуары (статусные товары).
6. Просчитать необходимое количество (для клиентки — сколько докупить).

**3 ошибки (из курса, для shopping-list как анти-правила):**
1. Слишком много брендов в одной ценовой категории — клиентке проще, когда 2–3 опорных бренда на сегмент.
2. Похожие вещи из разных категорий рядом — дублирование роли, ломает связность.
3. Нет аналитики потребности — покупка без привязки к пробелу в капсуле.

## Как подключается

- `prompts/shopping-list.md` (TODO): на вход — образы из `look-generator` + `price_segment` + поля Формулы. Фильтрует бренды этой матрицы по сегменту и полю, цвет/силуэт проверяет по `colortypes`/`figure-correction` на уровне вещи, выдаёт «вещь + бренд + поисковый запрос». Приоритет вещей — по группам товаров (локомотивы/комплексные первыми).

- Фаза 2 (живой каталог Lamoda/WB/Ozon) заменит «поисковый запрос» на реальные SKU — матрица брендов останется как фильтр доверенных марок.

## Очередь на валидацию Ксенией

1. Состав брендов: кого убрать (ушёл с рынка / не её уровень), кого добавить (её проверенные на клиентках).
2. Привязка к полям — спорные: Lime (natural+classic?), Lichi (drama?), 12 Storeez (Quiet Luxury = classic+natural?).
3. Региональное наличие (онлайн-доставка vs офлайн в её городе).
4. Бренды для пограничных стилей: Quiet Luxury (classic+natural), Power Woman (classic+drama) — кто лучший носитель.
