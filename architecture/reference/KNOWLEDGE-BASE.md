# Карта базы знаний — покрытие и конвейер

Единый учёт источников для AI-слоя: что уже превращено в структурную базу, что в очереди, что относится к бизнесу фаундера (не к движку). Обновляется при каждом разборе.

> **Архитектура слоя** (эвергрин/тренды, Vision-квиз, шопинг) — [`../2026-06-09-knowledge-and-vision-architecture.md`](../2026-06-09-knowledge-and-vision-architecture.md).
>
> **Два слоя по сроку годности:** эвергрин-метод (цвет/фигура/принты/типажи/психология) versus актуальное (`trends/` — сезоны, новости, стили 2026-2027). Тренд не перебивает эвергрин.

## Конвейер (как источник становится знанием)

```
raw-inbox/ (исходник: PDF, скрин, аудио/видео-транскрипт)
   → разбор: выделяем, что влияет на генерацию клиенту
   → структурный .md в нужную базу (или новая база)
   → пойнтер из промпта (formula-diagnostic / look-generator / style-book)
   → отметка в этой карте (статус ✅)
```

**Два потока источников:**
- **ДВИЖОК** — влияет на то, что AI генерирует клиентке (цвет, фигура, стиль, капсула, тексты). Идёт в `architecture/reference/` как структурный `.md`.
- **БИЗНЕС/фаундер** — про ремесло стилиста, ценообразование, ЦА, этику, обучение. Идёт в `.business/`, не в движок.

**Аудио/видео встреч (будущее):** транскрипт класть в `raw-inbox/transcripts/` с именем `YYYY-MM-DD-тема.md` (текст расшифровки). Дальше — тот же конвейер: выделяем «движковое» → в структурную базу. Сырой транскрипт не подключается к промптам напрямую, только его структурный вывод.

## Приоритет очереди (движок)

1. **Стилевые типажи** (Модуль 4) — ядро Формулы стиля, сейчас в `style-library` без глубокого источника.
2. **Психология/диагностика запроса** — усиливает `formula-diagnostic` (типы запросов, проблемное интервью).
3. **AI-генерация образов** — нужна для `preview_look` (WOW-якорь квиза).
4. **Тренды** — обновляемая база, чтобы капсулы были актуальны.
5. Остальное (конституция тела, ассортиментная матрица, домашний гардероб) — по мере надобности.
