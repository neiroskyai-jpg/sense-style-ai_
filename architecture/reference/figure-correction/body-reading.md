# Чтение конституции тела по фото — модульная схема пропорций

Как AI (Vision) снимает пропорции фигуры с фото без сантиметра и определяет тип. Дополняет `figure-correction.md` (там — что делать с типом, здесь — КАК определить тип и особенности). Источники: «Анализ конституции тела» + «Схема снятия плоскостных и объёмных характеристик» (курс «Алгоритмы имиджа»).

## Ключевая идея для Vision

Вся схема построена на **относительных пропорциях** (доли роста), а не на абсолютных сантиметрах — поэтому она работает по фото: измеряем в «высотах головы» или долях роста по пикселям. Нужно фото в облегающей одежде, в полный рост, в трёх ракурсах: **анфас, профиль, со спины** (минимум — анфас + профиль).

## Алгоритм анализа (5 шагов курса)

1. Субъективная (визуальная) оценка: что бросается в глаза (плечи vs бёдра, длина ног, посадка груди).
2. Снятие плоскостных и объёмных характеристик по модульной схеме.
3. Выявление реальных отклонений от эталона.
4. Сравнение с типовыми телосложениями.
5. Вывод: тип фигуры + индивидуальные особенности, требующие коррекции.

## Антропометрические точки

| Точка | Что это |
|---|---|
| A | темя |
| C | конец подбородка |
| C1 | основание шеи (CC1 = длина шеи) |
| D | наиболее выступающая точка груди |
| E | линия талии |
| F | паховая точка (начало ног) |
| J | точка под коленной чашечкой |
| K | пол (стопы) |

**Модуль H = высота головы (AC) = 1/8 роста.** Эталон: голова укладывается в рост 8 раз.

## Эталонные вертикальные пропорции

| Отрезок | Что измеряет | Норма | Отклонение и что оно значит |
|---|---|---|---|
| AC | голова | 1/8 h | 1/7 h — голова крупная; 1/9 h — маленькая. Диапазон 7,5–8,5 «голов в росте» — норма, не корректируется |
| AC1 | темя → основание шеи | 1/6 h | факт больше — шея длинная; меньше — короткая (допуск ~0,5 см) |
| AD | темя → точка груди | 1/4 h | факт больше — грудь посажена низко, корректировать в первую очередь: от уровня груди зависит визуальная длина ног |
| AE | темя → талия | 3H (3/8 роста) | факт меньше — талия завышена (торс выглядит коротким, ноги длинными); больше — занижена |
| AF | темя → пах (туловище) | 1/2 h | AF и FK должны быть равны — баланс торс/ноги |
| FK | пах → пол (ноги) | 1/2 h | FK < AF — ноги короче торса (типично для нижнего треугольника) |
| FJ / JK | бедро / голень | J делит ногу пополам | в жизни редко; у большинства голень длиннее бедра — это плюс (см. правила) |

## Эталонные плоскостные и объёмные характеристики

| Характеристика | Норма |
|---|---|
| Ширина плеч | 1/4 роста |
| Ширина груди | 1/6 роста |
| Ширина бёдер | = ширина плеч (плечи шире на 0–2 см — сбалансировано) |
| Обхват шеи | = обхват икры |
| Размер груди | обхват груди − обхват под грудью: норма > 8 и < 18 см |
| Выраженность талии | обхват бёдер − обхват талии |

## Определение типа по снятым характеристикам

- **Бёдра шире плеч** → нижний треугольник (pear). Ноги короче торса для этого типа — нормальное явление.
- **Плечи и бёдра сбалансированы (плечи шире на 0–2 см) + ОБ − ОТ ≥ 25 см** → песочные часы (hourglass).
- **Плечи заметно шире бёдер** → перевёрнутый треугольник (inverted_triangle).
- **Баланс плеч/бёдер есть, но ОБ − ОТ < 25 см** (талия не выражена) → прямоугольник (rectangle).
- Объём, сосредоточенный в середине (талия ≥ бёдер визуально) → apple.

Дальше тип идёт в `figure-correction.md` (приёмы коррекции по 5 типам). По фото обхваты не измерить точно — Vision оценивает выраженность талии визуально в анфас (изгиб бокового контура) и помечает уверенность.

**Профильный баланс:** в профиль грудь и ягодицы должны выступать одинаково относительно вертикальной боковой оси. Неравномерно → дисбаланс, уравновешивать (даже при нормальном размере груди).

**Экспресс-проверка длины рук:** уровень локтя должен совпадать с линией талии.

## Правила коррекции по отклонениям (из источника)

| Отклонение | Правило |
|---|---|
| Голова крупнее нормы | не увеличивать: без пышных причёсок; тёмные волосы уменьшают объём головы, светлые — увеличивают |
| Крупная голова при нормальной шее | шею визуально не укорачивать |
| Грудь посажена низко | сначала бельё (подтянуть лямки до уровня 1/4 h); если бельём не выходит — визуальная коррекция одеждой. Поднятая грудь визуально удлиняет ноги |
| Профильный дисбаланс (грудь < ягодицы) | уравновешивать профиль объёмом/акцентами сверху |
| Талия завышена | торс выглядит коротким — учитывать при выборе посадки, не задирать линию талии ещё выше |
| Торс длинный, ноги короткие | талию оставить на природном месте, ноги удлинять визуально (см. `figure-correction.md` §4) |
| Голень длиннее бедра | можно носить длину ниже колена, не укорачивая ноги, — при условии, что талия на месте |
| Короткие ноги + низкое бедро | юбка на природной талии: в юбке не видно, откуда начинаются ноги (в отличие от брюк) — ноги выглядят нормальной длины |

## Контрольные кейсы (валидация логики)

- **Ирина (159 см)** — песочные часы: плечи шире бёдер на 1,5 см, ОБ − ОТ = 26,5; AF = FK = 79,5 (идеальный баланс); грудь низко (45 при норме 39,75) → коррекция бельём; талия завышена на 2,5 см; голова 1/7,4 роста — крупновата, но в допуске.
- **Татьяна (163,5 см)** — нижний треугольник: бёдра шире плеч на 6 см; торс длинный (85 при норме 81,75), ноги короткие (79,5); шея длинная (+1,2 см); грудь низко (48,5 при норме 40,9); голень 45,5 против бедра 34 → юбка на природной талии решает длину ног.

## Как подключается

- `formula-diagnostic` ШАГ 7 («Силуэт») и анкета: пропорции читаются по этой схеме, коррекция — по `figure-correction.md`.
- Будущий `vision-analyzer.md`: этот файл — ядро инструкции «что читать с фото» по фигуре (порядок: рост в головах → вертикали → ширины → профиль → тип).
- Язык для клиентки — только «пропорции», без «тип фигуры»/«недостатки» (см. «Бережный язык» в `formula-diagnostic`).
