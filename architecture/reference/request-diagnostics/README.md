# architecture/reference/request-diagnostics/

База диагностики запроса — knowledge base для AI-слоя. Как читать запрос клиентки и приводить его к конструктивному до расчёта Формулы и Identity Gap.

| Файл | Что внутри | Статус |
|---|---|---|
| `request-diagnostics.md` | Конструктивный vs неконструктивный запрос (нереалистичный/неопределённый/манипулятивный), правило переформулирования, Точка А/Б | ✅ v1.0, требует валидации |

## Назначение

`formula-diagnostic` до расчёта приводит запрос к конструктивному (про себя, свои чувства). Манипулятивные «хочу, чтобы муж/мама/коллеги…» → «хочу чувствовать…». Это требование Self-Discrepancy: разрыв меряется между Я-реальным и Я-идеальным самой клиентки.

## Источник

«Типы запросов у клиентов», «Проблемное интервью» («Алгоритмы имиджа»). «Сложные клиенты» из того же кластера — операционка стилиста, ушла в `.business/`.
