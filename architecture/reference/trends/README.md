# architecture/reference/trends/

Актуальный слой базы знаний — тренды со сроком годности. В отличие от эвергрина (цвет/фигура/принты/типажи), портится по сезонам. Архитектура слоя — `../../2026-06-09-knowledge-and-vision-architecture.md`.

## Принцип

**Тренд не перебивает эвергрин.** При сборке образа цвет и фигура — фильтр, тренд — акцент внутри фильтра. Модное ≠ подходящее.

## Формат записи тренда (обязательные поля)

```
### <Название тренда>
- Сезон/срок: <FW2026 · действует до весны 2027>
- Кому по фигуре: <типы, кому работает / кому нет>
- Палитра / поле впечатления: <в какой палитре, к какой грани Формулы>
- Как применить: <конкретно — куда ставить, с чем, в каком объёме>
- Источник: <курс / встреча / веб, дата>
```

Без привязки «кому по фигуре + в какой палитре + как применить» тренд в базу не идёт — иначе `look-generator` не сможет его отфильтровать.

## Гибридное обновление

Твои источники (курс, встречи, подборки) в `../raw-inbox/` — основа; мой веб-поиск новостей моды — дополнение. Всё проходит через твоё подтверждение перед выкаткой. Устаревшее уезжает в `archive/`.
