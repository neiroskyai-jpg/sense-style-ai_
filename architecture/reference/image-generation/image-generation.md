# Генерация образов — фотореалистичный preview

**Тип:** knowledge base для AI-слоя. Как собирать `preview_look.image_generation_prompt` (английский промпт для image-API), чтобы получить фотореалистичный образ клиентки в её Формуле стиля. Это WOW-якорь квиза (2 бесплатных фото).

**Источник:** «Структура промпта. Параметры и команды», «Чек-лист по фотореалистичным генерациям» школы «Алгоритмы имиджа» (raw-inbox). Инструменты курса — Midjourney/SYNTX; принципы тул-агностичны (переносятся на любой image-API нашего стека).

**Статус:** v1.0, требует валидации Ксенией.

---

## 1. Что генерируем

Не «модную картинку», а **клиентку в её Формуле стиля**: её лицо и фигуру (из загруженного фото), одетые по визуальной формуле, в сценарии из её жизни, в фотореалистичном качестве. Образ должен закрывать разрыв идентичности — показать «Точку Б».

**Сохранение идентичности:** лицо и фигура клиентки сохраняются (фото как reference / image-to-image). Генерация меняет одежду и контекст, не внешность.

## 2. Структура промпта (6 блоков)

```
[субъект + идентичность] + [одежда из Формулы] + [сцена/роль] +
[фотостиль: камера/плёнка/свет] + [реализм/качество] + [negative/стоп]
```

1. **Субъект + идентичность:** woman, [возрастной код], + reference на фото клиентки (лицо/фигура сохранить).
2. **Одежда из Формулы:** силуэт + ткани + цвета палитры + деталь подстиля (из `../../sense-style-method.md` и `colortypes`/`figure-correction`). Только цвета её палитры — табу-цвета не генерируем.
3. **Сцена/роль:** контекст из её жизни (деловая встреча, ужин, прогулка) — из этапа «Контекст» метода.
4. **Фотостиль:** камера/плёнка + свет + журнальный стиль (раздел 3).
5. **Реализм:** `hyperrealistic photography`, `real fashion photo`, `editorial photoshoot`, `model portrait`.
6. **Negative/стоп:** что не должно появиться (стоп-цвета, стоп-силуэты из Формулы; искажения лица/рук).

## 3. Фотореалистичный тулкит

**Маркеры реализма (с любым параметром):** `hyperrealistic photography`, `real fashion photo`, `professional photography`, `model portrait`, `vogue style`, `editorial photoshoot`.

**Профессиональные камеры (резкость, контраст):** `shot on Hasselblad X1D` (фэшн-эталон), Canon EOS R6 Mark II, FUJIFILM XT5, Canon PowerShot G7 X.

**Плёнки (теплота, кожа):**
- `Agfa Vista Plus 200` — естественный тон кожи, портрет.
- `Kodak Portra` — природа, лёгкая дымка, мягкая кожа.
- `Fujifilm Pro 400H` — дневной свет, рекламные/модельные.
- `Kodak Gold 200` — насыщенность, мелкое зерно, резкость.
- `Kodak Vision3 500` / `golden hour ambience` — золотой час.
- `Kodak Ektar` — ретро, контраст.
- `Tri-X 400` — чёрно-белое, низкий свет.

**Ретро (для casual/молодёжных):** `Lomo`, `Disposable Camera` — высокий контраст, эффект 2000-х.

**Стиль/референс:** `in style of Helmut Newton photography`, `Vogue style`, `GQ magazine style`. + уточнения по свету, погоде, цветовой палитре.

## 4. Маппинг Формулы → фотостиль

| Формула / подстиль | Фотостиль образа |
|---|---|
| Power Woman / Минимализм / Драма-классика | editorial, Vogue, Hasselblad, чистый студийный/архитектурный фон, чёткий свет |
| Романтика / Леди-лайк | Kodak Portra, мягкий свет, golden hour, природа/интерьер |
| Натуральный / Smart casual / Trendy-mama | Fujifilm/Agfa, дневной естественный свет, городская улица |
| Спорт-шик / casual молодой | Lomo / GoPro, движение в кадре, ретро |

Палитра кадра (фон, свет) согласуется с цветотипом: тёплой осени — тёплый золотой свет; холодному лету — прохладный мягкий.

## 5. Параметры (Midjourney/SYNTX — только как учебный ориентир)

`/imagine` text-to-image; `--ar` соотношение сторон (портрет 3:4 / 46:61); `/blend` или ссылки с весами `{link::2, link::1}` для микса; `/describe` image-to-text (обратный разбор референса). **Midjourney — это инструмент КУРСА, не продакшена** (Discord/обёртки, нет чистого API). Принципы промпта (раздел 2-4) переносятся на любой image-API; синтаксис параметров — свой у каждого.

## 5a. Production-стек (решение 2026-06-09)

Два РАЗНЫХ этапа, не путать:

| Этап | Модель | Стоимость | Почему |
|---|---|---|---|
| **Анализ фото** (цветотип/фигура/контраст) | **Claude Vision** (наш стек) | ~<1 ₽/снимок | один провайдер, выход идёт прямо в рассуждение Claude |
| **Генерация preview** (2 образа клиентки) | **Gemini 2.5 Flash Image («Nano Banana»)** | ~1-3 ₽/изобр. | image-to-image сохраняет лицо, дёшево, фотореализм. Для тонкой задачи — возможно версия дороже |

- **Midjourney в продакшене НЕ используем** (учебный инструмент).
- **Почему Nano Banana, а не OpenAI gpt-image-1:** решает сохранение идентичности (наш preview = «это Я в новом образе», image-to-image с лицом клиентки). Nano Banana держит того же человека при смене одежды лучше и дешевле (~1-3 ₽ vs ~2-15 ₽). gpt-image-1 сильнее в тексте-в-кадре и сцене с нуля — нам это не критично. Перед фиксацией — A/B на 2-3 реальных селфи (похожесть субъективна).
- **Картинка ≠ спецификация.** Image-модели не контролируют точные цвета палитры и силуэт. Сгенерированный образ продаёт ОЩУЩЕНИЕ направления; точную палитру (без табу), силуэт и вещи несёт текст (Style Book / капсула / шопинг-лист).
- Через **OpenRouter** держим Claude (текст+vision) + Gemini (картинки) под одним API.
- Итого на бесплатную анкету: ~3-7 ₽ (анализ + 2 картинки) — preview экономически проходит.

**Операционные правила (из практики смежного проекта):**
1. **Сжимать фото на входе** перед Vision — большие изображения = долго и дорого.
2. **Генерировать preview асинхронно** — формулу/анализ показываем сразу, картинки догружаются (латентность генерации — самый медленный шаг).
3. **Тестировать каждый компонент отдельно в чате, потом склеивать** — не строить и интегрировать параллельно.

Источник практики — смежный проект (конструктор объявлений недвижимости): тот же стек image-to-image на Nano Banana, распознавание на GPT/Claude, текст на Sonnet через OpenRouter.

## 6. Как подключается

`formula-diagnostic` заполняет `preview_look.image_generation_prompt` по структуре раздела 2: субъект+идентичность клиентки → одежда из её Формулы (палитра без табу, силуэт по фигуре) → сцена из её роли → фотостиль по подстилю (раздел 4) → маркеры реализма → negative. Бесплатный preview = 2 образа (главное направление + нота), как в mvp-spec.

> При расхождении с подходом Ксении — выигрывает Ксения; справочник правится.
